# 飞书接收报告与看板链接 - 具体操作步骤

按下面步骤做完后，可以在飞书里定期收到营销报告，或随时点链接打开看板。

---

## 一、前置：飞书机器人与 .env（只需做一次）

### 1. 在飞书里创建机器人

1. 打开飞书 → 进入要接收报告的**群聊**
2. 点击群设置（右上角 ⋮）→ **群机器人** → **添加机器人** → 选 **自定义机器人**
3. 填写名称（如「好特卖看板」），可选勾选「启用签名」；完成后会得到一个 **Webhook 地址**，形如：  
   `https://open.feishu.cn/open-apis/bot/v2/hook/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx`
4. 复制该地址备用

### 2. 在项目里配置 Webhook

在项目根目录的 **`.env`** 里增加或修改（把下面的地址换成你复制的）：

```bash
# 飞书机器人 Webhook（必填，报告会发到这个群）
FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/你的hook-id

# 可选：@ 指定人（填接收人的 open_id，可在飞书管理后台-通讯录-用户详情里看）
# FEISHU_AT_USER_ID=ou_xxxxxxxx
# FEISHU_AT_USER_NAME=余为军
```

保存后，**重启看板服务**（若正在运行）：  
`npm run htma:run` 或 `bash scripts/start_htma.sh`  
这样后续用 API 或定时任务推送时，都会发到这个群。

---

## 二、用 API 把报告推到飞书（手动触发）

看板服务需已启动（`npm run htma:run`），然后任选一种方式。

### 方式 A：浏览器里点链接（最省事）

1. 在浏览器地址栏输入（本机看板）：

   ```
   http://127.0.0.1:5002/api/marketing_report?send=1
   ```

2. 若看板在**服务器**上，把 `127.0.0.1` 换成服务器 IP 或域名，例如：

   ```
   http://192.168.1.100:5002/api/marketing_report?send=1
   ```

3. 回车后，页面会返回一段 JSON；同时**报告会发到你在 .env 里配置的飞书群**。

### 方式 B：命令行（本机或 SSH 到服务器执行）

```bash
# 进入项目目录
cd /Volumes/ragflow/hotmaxx/hotmaxxflag

# 带 .env 并请求“生成并推送”
source .env 2>/dev/null; curl -s "http://127.0.0.1:5002/api/marketing_report?send=1"
```

若看板在服务器上，把 `127.0.0.1` 改成服务器 IP，例如：

```bash
curl -s "http://你的服务器IP:5002/api/marketing_report?send=1"
```

### 方式 C：用项目自带脚本（内部会调 API 或直接发飞书）

```bash
cd /Volumes/ragflow/hotmaxx/hotmaxxflag
npm run htma:report
# 或
.venv/bin/python scripts/htma_marketing_report.py
```

以上三种都会把**进销存营销分析报告**推到飞书；区别只是触发方式（浏览器 / curl / 脚本）。

## 三、定时任务：按固定节奏自动推到飞书

用系统自带的 **crontab**，让报告每天固定时间发到飞书，无需每次手动点。

### 1. 打开 crontab

在终端执行：

```bash
crontab -e
```

（首次会提示选编辑器，选 nano 或 vim 即可。）

### 2. 添加一行定时任务

在文件**末尾**加一行（以下按「每天 9 点」举例，路径请按你本机实际改）：

```bash
# 每天 9:00 生成进销存营销报告并推送到飞书
0 9 * * * cd /Volumes/ragflow/hotmaxx/hotmaxxflag && .venv/bin/python scripts/htma_marketing_report.py >> /tmp/htma_report.log 2>&1
```

说明：

- `0 9 * * *`：每天 9:00 执行  
  若想改成 18:00：`0 18 * * *`  
  若想工作日 9:00：`0 9 * * 1-5`
- 路径 `/Volumes/ragflow/hotmaxx/hotmaxxflag` 若不同，改成你项目所在路径。
- 日志在 `/tmp/htma_report.log`，出错时可 `cat /tmp/htma_report.log` 查看。

### 3. 保存并生效

- **nano**：`Ctrl+O` 回车保存，`Ctrl+X` 退出  
- **vim**：按 `Esc`，输入 `:wq` 回车  

保存后定时任务即生效，到点会自动跑脚本并把报告推到飞书。

### 4. 注意：看板服务与 MySQL

- 脚本 `htma_marketing_report.py` 会连 **MySQL** 和（若用 API）**看板服务**；若报告是通过「调 API」生成的，则到点时**看板服务也要在运行**。
- 若你用的是「脚本直接生成并发飞书」（如上面 crontab 里的命令），则只要求 **MySQL 在运行**，不强制看板 24 小时开着。
- 若希望「到点自动调 API 再发飞书」，可以改成：

```bash
0 9 * * * curl -s "http://127.0.0.1:5002/api/marketing_report?send=1" >> /tmp/htma_report.log 2>&1
```

这时 9 点时看板服务必须已启动（例如同一台机子上看板用 systemd/supervisor 常驻）。

## 四、在飞书里放看板链接（点链接打开完整看板）

报告是摘要；要看完整看板（图表、明细、筛选），可以在飞书里放一个链接，需要时点开用浏览器看。

### 1. 确定看板地址

- **本机访问**：`http://127.0.0.1:5002`  
- **同一局域网其他人访问**：`http://你的电脑IP:5002`（如 `http://192.168.1.100:5002`）  
- **看板在服务器**：`http://服务器IP或域名:5002`（若做了域名或 Nginx 反向代理，则用你的实际访问地址）

端口默认 5002；若在 `.env` 里设过 `PORT=其他端口`，则用对应端口。

### 2. 在飞书里加链接的几种方式

**方式 A：发到群聊**

1. 在飞书群输入框里输入或粘贴看板地址，例如：  
   `http://192.168.1.100:5002`
2. 飞书通常会自动变成可点击的链接；若没有，可选中文字 → 右键 → **添加链接**，URL 填同上。

**方式 B：写在飞书文档里**

1. 新建或打开一篇飞书文档  
2. 输入说明文字，如：「好特卖运营看板：」  
3. 下一行粘贴看板地址，或选中文字后点工具栏的「链接」图标，把看板地址填进去  
4. 保存文档；之后需要看数据时，在飞书里打开文档点链接即可用浏览器打开看板

**方式 C：放在机器人/群公告里**

- 若群里有「群公告」或「群介绍」，可把看板链接写进去  
- 或让机器人固定回复里带这条链接（若你做了自定义机器人逻辑）

这样，**报告** = 飞书里定期/手动推送的摘要；**完整数据** = 需要时在飞书里点链接打开浏览器看板。

---

## 五、小结对照表

| 目的                     | 做法 |
|--------------------------|------|
| 手动发一次报告到飞书     | 浏览器打开 `http://看板地址:5002/api/marketing_report?send=1` 或执行 `npm run htma:report` |
| 每天固定时间自动发报告   | `crontab -e` 添加 9 点的 `htma_marketing_report.py` 或 curl `...?send=1` |
| 在飞书里随时看完整看板   | 在群消息/文档/公告里加入看板链接（如 `http://服务器:5002`），需要时点击打开 |

按上面顺序做完「一、前置」和「二」或「三」，就可以用 API 或定时任务把报告推到飞书；按「四」把看板地址加到飞书文档/消息里，即可在飞书里点链接打开完整看板。
