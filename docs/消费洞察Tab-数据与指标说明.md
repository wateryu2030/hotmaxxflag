# 消费洞察 Tab · 数据与指标说明

## 一、现有数据情况

| 数据源 | 可用字段（与消费洞察相关） |
|--------|---------------------------|
| **t_htma_sale** | data_date, sku_code, store_id, sale_qty, sale_amount, sale_cost, gross_profit, category / category_large / category_mid / category_small, product_name, brand_name, supplier_name, sale_price, return_qty, return_amount, gender, style, division, color_system, biz_mode |
| **t_htma_profit** | data_date, category, total_sale, total_profit, profit_rate |
| **t_htma_stock** | data_date, sku_code, stock_qty, stock_amount, product_name, brand_name |
| **t_htma_product_master** | sku_code, store_id, product_status(新品/正常), retail_price, list_price, member_price, distribution_mode(购销/代销), category_name, brand_name, supplier_name, archive_date |

**缺失或不可用**：订单数/交易笔数（无法算真实客单价）、进货数量（无法算售罄率）、临期/过期日期（需批次表才能算临期占比）。

## 二、需求与实现对照（可落地部分）

### 1. 核心经营指标（整体健康度）

| 指标 | 实现方式 | 说明 |
|------|----------|------|
| 总销售额 | SUM(sale_amount) | 直接实现 |
| 总毛利额 | SUM(gross_profit) | 直接实现 |
| 毛利率 | 毛利额/销售额×100% | 直接实现 |
| 动销商品数 | COUNT(DISTINCT sku_code) | 直接实现 |
| 动销率 | 动销商品数 / 商品主数据总数 | 需关联 t_htma_product_master 或 t_htma_products 总数（同一 store_id） |
| 客单价 | — | **不做**：无订单数 |
| 件单价 | 销售额/销售数量 | 直接实现 |

### 2. 商品贡献分析

| 维度 | 实现方式 |
|------|----------|
| 品类 | 销售额 Top10、毛利额 Top10、毛利率 Top5（按 category 或 category_large/category_mid） |
| 品牌 | 品牌销售额占比、毛利贡献、平均毛利率（brand_name 来自 sale） |
| 单品 | 单品销售额排名、单品毛利率（销售明细已有；可加周转需关联库存） |
| 价格带 | 各价格带销售额/销量占比（用 sale 的 sale_price 或 product_master.retail_price 分段） |
| 色系/风格 | 有则展示（sale 有 color_system, style）；无则隐藏 |

### 3. 价格策略与毛利

| 指标 | 实现方式 |
|------|----------|
| 平均折扣率 | 1 - AVG(实际售价/划线价)，需 sale 关联 product_master.list_price |
| 价格带渗透率 | 各价格带销量/总销量 |
| 毛利率×价格带 | 按价格带汇总毛利率与销售额（气泡图数据） |
| 会员价使用率 | **暂不做**：需订单级“是否使用会员价” |
| 最低售价偏离度 | **可选**：product_master 有 min_sale_price 时可算 |

### 4. 供应链与经销模式

| 维度 | 实现方式 |
|------|----------|
| 经销方式 | 代销 vs 购销：销售额占比、毛利率（sale 关联 product_master.distribution_mode） |
| 供应商 | 供应商销售额/毛利排名、动销 SKU 数（sale 有 supplier_name） |
| 库存效率/售罄率 | **不做**：无进货数量 |
| 临期/保质期 | **不做**：无批次/到期日 |

### 5. 新品与生命周期

| 指标 | 实现方式 |
|------|----------|
| 新品销售额占比 | product_status=新品 的 sale 汇总 / 总销售额（关联 product_master） |
| 新品动销率 | 有销售的新品 SKU 数 / 新品总 SKU 数（product_master 中 status=新品） |
| 新品毛利率 | 新品毛利/新品销售额 |
| 老品清仓 | **简化**：老品（非新品）销售额占比、平均折扣（有 list_price 时） |

### 6. 趋势与异常

| 指标 | 实现方式 |
|------|----------|
| 销售额日/周环比 | 复用现有走势接口或按日/周汇总对比 |
| 退货率 | 按品类/品牌：return_amount/销售额 或 return_qty/销量（sale 有 return_qty/return_amount） |

## 三、消费洞察 Tab 落地范围（首版）

- **概览 KPI**：总销售额、总毛利、毛利率、动销商品数、件单价；可选动销率（需商品主数据总数）。
- **品类贡献**：销售额 Top10、毛利 Top10、毛利率 Top5（与经营分析品类结构互补）。
- **品牌贡献**：品牌销售额、毛利、销量、毛利率、贡献占比（表格）。
- **价格带**：各价格带销售额/销量占比、毛利率（与经营分析价格带一致或复用接口）。
- **经销方式**：代销 vs 购销 销售额占比、毛利占比、毛利率（关联 product_master.distribution_mode）。
- **新品表现**：新品销售额占比、新品动销率、新品毛利率（关联 product_master.product_status）。
- **退货率**：按品类或品牌汇总退货额/退货量占比（可选，经营分析已有退货/赠送）。

数据周期与 KPI 周期一致（start_date / end_date），与经营分析、税率计算同源同周期。

## 四、与原始 BI 指标体系对照（缺项与实现状态）

| 文档章节 | 指标/维度 | 实现状态 | 说明 |
|----------|-----------|----------|------|
| **一、核心经营指标** | 总销售额、总毛利额、毛利率、动销商品数、件单价、动销率 | ✅ 已实现 | 客单价因无订单数不做 |
| **二、商品贡献** | 品类销售额Top10、毛利额Top10、毛利率Top5 | ✅ 已实现 | 三张表分别展示 |
| | 品牌销售额占比、毛利贡献、毛利率 | ✅ 已实现 | 品牌贡献表 |
| | 单品销售额排名、单品毛利率 | ✅ 已实现 | 单品销售 Top20 表；周转速度需库存、可后续加 |
| | 价格带销售额占比、各价格带毛利率、**价格带渗透率（销量占比）** | ✅ 已实现 | 价格带表含销量占比% |
| | 色系/风格销售额、毛利率 | ✅ 已实现 | 色系表、风格表（有字段则展示） |
| **三、价格策略** | 平均折扣率（1-实际售价/划线价） | ✅ 已实现 | 关联 product_master.list_price，概览 KPI 展示 |
| | 价格带渗透率、毛利率×价格带 | ✅ 已实现 | 同上价格带表 |
| | 会员价使用率、最低售价偏离度 | ❌ 不做 | 需订单级会员价、min_sale_price 等 |
| **四、供应链/经销** | 经销方式（代销 vs 购销）销售额、毛利率、占比 | ✅ 已实现 | 经销方式表 |
| | 供应商贡献排名、动销 | ✅ 已实现 | 供应商贡献表（销售额、毛利、销量、毛利率、贡献%） |
| | 库存效率/售罄率、临期/保质期 | ❌ 不做 | 无进货数量、无批次/到期日 |
| **五、新品与生命周期** | 新品销售额占比、新品动销率、新品毛利率 | ✅ 已实现 | 新品表现表 |
| | 老品销售占比 | ✅ 已实现 | 新品表现表中「老品销售占比」行 |
| | 老品清仓效率（降幅、折扣率） | ⚠️ 简化 | 仅老品占比；精细清仓需历史对比 |
| **六、趋势与异常** | 销售额环比（本期 vs 上期同长度） | ✅ 已实现 | 概览「环比」KPI（近30天 vs 前30天） |
| | 退货率（整体 + 按品类） | ✅ 已实现 | 概览「退货率」KPI + 退货率按品类表 |
| | 价格敏感度、滞销/临期预警 | ❌ 不做 | 需历史调价数据、库存/批次 |
| **七、可视化** | 概览 KPI 卡片、品类/品牌/价格带表、经销/供应商、新品、退货 | ✅ 已实现 | 与文档建议一致；散点图/折线图可后续加 |
| **八、后续分析** | 关联分析、价格弹性、库存健康度矩阵 | ❌ 不做 | 需订单/购物篮、历史价量、进货与库存 |

**小结**：在现有数据（销售日报、毛利、商品主数据）下，文档中可落地的指标已全部在消费洞察 Tab 实现；仅依赖订单数、进货、批次/到期日、会员价等字段的项标注为「不做」或「简化」。

## 五、六大维度增强与数据可行性（BI 深度分析看板）

基于商品主数据、销售日报、毛利、库存表，以下为可落地项与不可做项对照。

| 设想模块 | 数据是否支持 | 落地方式 |
|----------|--------------|----------|
| **一、整体概览** | | |
| 总商品数、动销商品数、动销率、总销售额、总毛利额、毛利率、件单价 | ✅ 已有 | 保持 |
| 平均零售价 | ✅ product_master.retail_price | 新增：AVG(retail_price) |
| 平均折扣率 | ✅ 已有 | 保持 |
| 库存周转天数 | ✅ stock + sale | 新增：平均库存/日均销量 |
| **二、商品结构** | | |
| 品类贡献矩阵（SKU数、动销SKU、动销率、销售额、占比、毛利、毛利率、平均售价、平均折扣率） | ✅ 销售+主数据 | 合并三张品类表为一张矩阵表，补充动销SKU、平均售价、平均折扣率；条形图 Top10 品类销售额 |
| 品牌效能（SKU数、销售额、毛利、毛利率、平均折扣率、动销率） | ✅ 销售 | 品牌表补 sku_sold；新增散点/气泡图（SKU-销售额-毛利-毛利率） |
| 价格带（SKU数、SKU占比、销售额、占比、毛利、毛利率、平均折扣率） | ⚠️ 部分 | 价格带按销售已有；SKU数按零售价分箱需主数据，可补 |
| 属性交叉（性别、风格、色系） | ✅ sale 有 gender/style/color_system | 已有色系/风格表；可补性别饼图/条形图 |
| **三、价格与利润** | | |
| 折扣区间分析（0–10%、10–20% 等） | ✅ sale+list_price | 新增：按折扣区间汇总表 + 堆积柱状图 |
| 价格带×毛利率矩阵（四象限/散点） | ✅ 已有价格带+毛利率 | 新增：散点图（价格带 vs 毛利率，气泡=销售额） |
| 会员价使用率 | ❌ 无订单级会员价 | 不做 |
| **四、供应商与经销** | | |
| 供应商绩效（SKU数、动销率、销售额、毛利、毛利率、平均折扣率、退货率） | ✅ 销售有 supplier/return | 表可补退货率；雷达图可选 |
| 经销方式对比 | ✅ 已有 | 新增：饼图 + 对比柱状图 |
| 联营扣率 | ⚠️ 需合同数据 | 暂不做 |
| **五、新品与生命周期** | | |
| 新品表现（占比、动销率、毛利率、平均折扣率） | ✅ 已有；平均折扣率可补 | 保持并补新品平均折扣率 |
| 生命周期阶段（导入/成长/成熟/衰退） | ❌ 缺上架日期 | 用 archive_date 代理可做简化版，或不做 |
| **六、异常预警** | | |
| 零销售商品清单（周期内无销售） | ✅ product_master + sale | 新增：主数据中有、周期内无销售的 SKU 列表（限条数） |
| 临期商品预警 | ❌ 无到期日/批次 | 不做 |
| 高折扣低毛利商品 | ✅ sale+主数据 | 新增：折扣率>30% 且毛利率<10% 的 SKU/品类列表 |

**图表布局建议（与文档七对应）**：顶部 KPI 卡片 → 品类 Top10 条形图 + 品牌气泡图 → 价格带双轴图（柱+折线）→ 经销方式饼图 → 折扣区间堆积柱 → 品类贡献矩阵表（替代原三表）→ 零销售/高折扣低毛利预警表 → 供应商绩效表。
