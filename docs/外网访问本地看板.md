# 外网访问本地看板（本机在局域网内）

## 桌面图标一键启动（终端方式，不依赖 Docker）

项目里已准备好两个可双击的启动器，**先释放 5002 再启动**，避免“Address already in use”：

| 文件 | 作用 |
|------|------|
| **启动好特卖看板.command** | 仅启动看板，本机访问 http://127.0.0.1:5002 |
| **启动好特卖看板+外网访问.command** | 启动看板 + Cloudflare 隧道，终端里会显示外网 https 地址 |

**使用步骤：**

1. **放到桌面（可选）**：在项目目录执行一次  
   `bash scripts/创建桌面快捷方式.sh`  
   会在桌面生成两个图标；也可不复制，直接在项目里双击这两个 `.command` 文件。

2. **双击运行**：  
   - 只在本机看：双击「启动好特卖看板」。  
   - 需要外网访问：双击「启动好特卖看板+外网访问」，等终端里出现 `https://xxx.trycloudflare.com` 后，用手机或外网浏览器打开该链接。

3. **关闭**：关掉弹出的终端窗口，看板（和隧道）会一起停止。  
   若 5002 被占用（例如 Docker 或其他程序），启动器会先结束占用该端口的进程再启动。

---

看板当前在本地/局域网运行（如 `http://192.168.x.x:5002`），若需要在外网（公司外、家里、手机 4G 等）访问，可用下面两种方式之一。

---

## 方式一：内网穿透 / 隧道（推荐，不改路由器）

不配置路由器，在运行看板的机器上装一个「隧道」程序，把本机 5002 端口映射到公网，获得一个外网可访问的地址。

### 1. 用 Cloudflare Tunnel（免费、较稳定）

1. **安装 cloudflared**（任选一种）  
   - macOS（Homebrew）：`brew install cloudflared`  
   - 或到 https://developers.cloudflare.com/cloudflare-one/connections/connect-apps/install-and-setup/installation/ 按系统下载

2. **登录 Cloudflare（可选，不登录也能用快速隧道）**  
   ```bash
   cloudflared tunnel login
   ```
   不登录则用下面「快速隧道」一步到位。

3. **启动快速隧道（临时外网地址）**  
   在**运行看板的同一台机器**上执行：
   ```bash
   cloudflared tunnel --url http://127.0.0.1:5002
   ```
   终端会打印一行类似：
   ```text
   Your quick Tunnel has been created! Visit it at:
   https://xxxx-xx-xx-xx-xx.xx.trycloudflare.com
   ```
   用浏览器访问这个 `https://...trycloudflare.com` 即可在外网打开看板。  
   关闭终端或 Ctrl+C 后，该地址失效；下次再运行会得到新地址（若需固定域名，需用 Cloudflare 正式 Tunnel + 自定义域名）。

### 2. 用 ngrok（免费版有随机域名）

1. 注册 https://ngrok.com ，安装并配置 token  
2. 安装后执行：
   ```bash
   ngrok http 5002
   ```
3. 终端里会显示 `Forwarding` 的 https 地址，用该地址即可在外网访问看板。  
   免费版每次重启会变域名；付费可固定域名。

### 3. 用 frp（自建或别人提供的 frp 服务）

若你或公司有一台有公网 IP 的服务器，可在服务器上部署 frps、在本机部署 frpc，把本机 5002 映射到公网。配置示例（本机 frpc）：

```ini
[common]
server_addr = 你的公网服务器IP
server_port = 7000

[htma]
type = tcp
local_ip = 127.0.0.1
local_port = 5002
remote_port = 5002
```

服务器开放 7000（frps）、5002（访问看板）端口后，外网通过 `http://服务器IP:5002` 访问。

---

## 方式二：路由器端口转发（有公网 IP 时）

本机在家庭/公司路由器后面，且路由器有公网 IP 时，可在路由器上把「外网某端口」转发到「看板所在机器的 5002」。

### 1. 确认看板所在机器的内网 IP

在运行看板的机器上执行：

- macOS/Linux：`ifconfig | grep "inet "` 或 `ip addr`
- Windows：`ipconfig`

记下局域网 IP，例如 `192.168.1.100`。

### 2. 在路由器里做端口转发

1. 浏览器打开路由器管理页（常见为 `192.168.1.1` 或 `192.168.0.1`，见路由器背面说明）。
2. 登录后找到「端口转发 / 虚拟服务器 / NAT」等类似名称。
3. 新增一条规则，例如：
   - 外部端口：`5002`（或 80，若希望用 80 访问）
   - 内部 IP：`192.168.1.100`（看板所在机器）
   - 内部端口：`5002`
   - 协议：TCP（或 TCP/UDP）
4. 保存并生效。

### 3. 外网访问方式

- 若运营商给的是**固定公网 IP**：外网浏览器访问 `http://你的公网IP:5002`。
- 若公网 IP **会变**：可配 **DDNS**（如 No-IP、DuckDNS、花生壳或路由器自带 DDNS），用域名访问，例如 `http://你的域名:5002`。

---

## 安全提醒

- 当前看板**没有内置登录**，一旦暴露到外网，谁拿到地址都能看数据。
- 建议：
  - 尽量用 **https**（隧道方式本身多为 https；路由器方式可考虑用 Nginx 反代 + 证书，或仅在内网/VPN 用）。
  - 若必须长期外网开放：在路由器或 Nginx 层做** IP 白名单 / 简单认证**，或把看板放在 VPN 后，只通过 VPN 访问。

---

## 小结

| 场景 | 建议 |
|------|------|
| 临时、快速让外网能访问 | 本机执行 `cloudflared tunnel --url http://127.0.0.1:5002` 或 `ngrok http 5002`，用给出的 https 地址访问。 |
| 有公网服务器 | 用 frp 把本机 5002 映射到服务器，外网通过服务器 IP:5002 访问。 |
| 有公网 IP 且可改路由器 | 路由器端口转发 5002 到看板机器；IP 会变则配 DDNS 用域名访问。 |

看板已监听 `0.0.0.0:5002`，只要外网流量能到达本机 5002 端口（通过隧道或端口转发），即可在外网看到看板内容。
