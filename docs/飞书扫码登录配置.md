# 好特卖看板 - 飞书扫码登录配置

为保护敏感经营数据，看板支持「飞书扫码授权登录」：用户点击「飞书扫码登录」后跳转飞书授权页，使用飞书 App 扫码确认即可登录。

## 一、行为说明

- **未配置登录**：不设置 `FEISHU_APP_ID` / `FEISHU_APP_SECRET` 时，看板不启用登录，所有人可直接访问（与之前行为一致）。
- **已配置登录**：设置上述环境变量后，访问首页或数据导入页会先跳转到登录页，用户需通过飞书扫码授权后才能进入看板。
- **企业内/企业外**：通过飞书通讯录 API 判断用户是否为本企业成员。**企业内用户**直接登录；**企业外用户**扫码后会进入「等待审批」页，需由超级管理员在审批页**按用户**通过后，该用户再次登录即可访问（仅被审批通过的个人可访问，同一企业内未审批用户无法登录）。新申请时会在飞书群 @ 超级管理员 通知。**超级管理员专用**：审批入口 `https://htma.greatagain.com.cn/approval`；提醒页（待审批数量与快捷入口）`https://htma.greatagain.com.cn/admin`（仅超级管理员可访问）。超级管理员 open_id 可通过环境变量 `HTMA_SUPER_ADMIN_OPEN_ID` 配置（默认 8db735f2）。

## 二、飞书开放平台配置

1. **创建自建应用**
   - 打开 [飞书开放平台](https://open.feishu.cn/app) → 创建企业自建应用 → 填写名称等。

2. **获取凭证**
   - 在应用内进入「凭证与基础信息」→ 复制 **App ID**、**App Secret**。

3. **安全设置 - 重定向 URL（必填，否则会报 20029 redirect_uri 请求不合法）**
   - 在「安全设置」→「重定向 URL」中**逐条添加**下面用到的地址，**必须与后端使用的完全一致**（协议、域名、路径、无多余斜杠）：
     - **外网**：`https://htma.greatagain.com.cn/api/auth/feishu_callback`（不要写成 http、不要末尾加 `/`）
     - **本机开发**（可选）：`http://127.0.0.1:5002/api/auth/feishu_callback`
   - 在项目根目录 `.env` 中设置 **`HTMA_PUBLIC_URL=https://htma.greatagain.com.cn`**（不要带末尾斜杠），这样后端向飞书请求授权和兑换 code 时都会使用该地址，与飞书控制台里填的一致。
   - **本机开发**：若在飞书控制台同时添加了本机回调，从本机打开看板时登录后回调会回到本机；否则会跳到外网并保留 path+query。

4. **权限**
   - 在「权限管理」中为应用开通：**contact:user.base:readonly**（获取用户基础信息），用于登录后显示用户名，以及区分企业内/外用户（通过通讯录 API 查询用户是否在本企业）。

5. **应用可用性（解决「没有 hotmaxxblog 使用权限」）**
   - 若扫码时提示 **「你没有"hotmaxxblog"的使用权限」**、**「当前登录账号为 xxx(某公司)…无权限」**，说明是**飞书应用的使用范围**限制了谁能用该应用，与看板 .env 无关。
   - **处理步骤（须由飞书应用管理员在开放平台操作）：**
     1. 打开 [飞书开放平台](https://open.feishu.cn/app) → 使用**有管理权限的飞书账号**登录 → 进入应用 **hotmaxxblog**（App ID: cli_a9295df5c2789ceb）。
     2. 在左侧菜单找到 **「可用性」** 或 **「使用范围」** 或 **「应用发布」**（名称因控制台版本可能不同）。
     3. **给「外来」企业/申请者授权：** 在 **「可使用应用的企业」** 中，**添加允许其成员发起申请的企业**（例如：北京华锐光年文化发展有限公司、振鸿科技有限公司等）。未加入的企业下成员扫码会提示「无权限」；加入后，该企业成员**可以完成飞书扫码并进入看板的申请流程**。**注意：** 看板按**用户（个人）**审批，仅超级管理员在审批页点击「通过」的个别用户才能登录，**并非该企业全员可访问**；同一企业内其他未获审批的用户仍无法进入看板。
     4. **本企业内谁可用：** 查看 **「使用范围」/「可见范围」**，若为「仅部分成员」且未包含某人则无权限；可改为 **「全部员工」** 或勾选需要访问看板的所有部门/成员。
     5. 保存后等待约 1～2 分钟，再让申请者重新扫码登录。
   - **总结：** 要让某企业（含外来申请者）能**发起**看板登录申请，须在 hotmaxxblog 的**可用性 → 可使用应用的企业**中加入该企业；**谁实际能登录看板**由看板侧按**用户**审批决定，仅经超级管理员在审批页通过的个人可访问，本企业内则通过「使用范围」控制具体成员。

## 三、环境变量（.env）

在项目根目录 `.env` 中增加（或从 `.env.example` 复制后填写）：

```bash
# 飞书自建应用凭证（必填才启用登录）
FEISHU_APP_ID=你的App_ID
FEISHU_APP_SECRET=你的App_Secret

# 飞书回调与授权链接的站点地址（与飞书控制台「重定向 URL」一致；走 Cloudflare/代理时必填）
HTMA_PUBLIC_URL=https://htma.greatagain.com.cn

# 会话密钥（生产环境务必改为随机长字符串）
FLASK_SECRET_KEY=随机长字符串

# 可选：仅允许指定 open_id 登录（逗号分隔）。不填或留空则本企业内所有飞书用户均可登录（如振鸿科技群全员可见）
# HTMA_ALLOWED_FEISHU_OPEN_IDS=ou_xxx,ou_yyy
# 超级管理员 open_id（企业外用户审批人，默认余为军 8db735f2）。审批页仅该用户可访问。
# HTMA_SUPER_ADMIN_OPEN_ID=8db735f2
# 飞书机器人 webhook（用于新申请时 @ 余为军、功能上线通知等）
# FEISHU_WEBHOOK_URL=https://open.feishu.cn/open-apis/bot/v2/hook/xxx

# 使用 HTTPS 时建议开启（设了 HTMA_PUBLIC_URL 为 https 时会自动开启）
# SESSION_COOKIE_SECURE=1
```

获取用户的 **open_id**：飞书管理后台 → 通讯录 → 用户详情，或通过飞书 API 查询。

## 四、错误码 20029（redirect_uri 请求不合法）

若扫码或跳转飞书时出现 **错误码：20029 redirect_uri 请求不合法**，说明发给飞书的回调地址与开放平台里配置的「重定向 URL」不一致。按下面排查：

1. **确认 .env 中的公网地址**
   - 在项目根目录 `.env` 里必须有：**`HTMA_PUBLIC_URL=https://htma.greatagain.com.cn`**
   - 不要写 `http://`，不要末尾加 `/`。保存后**重启看板服务**（先停掉再 `npm run htma:run`）。

2. **在飞书开放平台添加「重定向 URL」**
   - 打开 [飞书开放平台](https://open.feishu.cn/app) → 进入你的应用（如 hotmaxxblog）→ **安全设置** → **重定向 URL**。
   - 点击「添加」，填入（一字不差）：  
     **`https://htma.greatagain.com.cn/api/auth/feishu_callback`**
   - 不要多空格、不要末尾斜杠、不要用 http。保存后等待约 1 分钟再重试登录。

3. **若仍报 20029**
   - 把当前使用的 redirect_uri 提供给客服（log ID 如 20260304142925D33EF216591F1146234B 已足够），或在本机查看：看板请求 `/api/auth/feishu_url` 时后端会用 `HTMA_PUBLIC_URL` + `/api/auth/feishu_callback` 拼成 redirect_uri，确认该字符串与飞书里填的**完全一致**。

## 五、验证

1. 配置完成后重启看板服务。
2. 浏览器访问看板首页（如 `http://127.0.0.1:5002`），应自动跳转到 `/login`。
3. 点击「飞书扫码登录」，跳转飞书授权页，用飞书 App 扫码并确认。
4. 授权成功后应跳回看板首页，右上角显示「欢迎，xxx 退出」。

## 六、企业微信登录（规划）

当前仅实现飞书扫码登录；企业微信扫码登录可后续按同样思路接入企业微信 OAuth，登录页已预留入口「企业微信扫码登录（敬请期待）」。

## 七、其他企业飞书用户申请访问（完整流程）

要让**非本企业**的飞书用户能访问看板（如 htma.greatagain.com.cn），需要同时满足：**飞书开放平台**允许该企业使用应用（以便其成员能完成扫码并进入申请流程），以及**看板侧按用户审批**（仅经超级管理员通过的个别用户可登录）。**重要：** 看板只允许**经审批的个别外部用户**访问，同一企业内未获审批的用户无法登录，并非加入企业后该企业全员可访问。

### 7.1 飞书开放平台侧（应用管理员操作）

1. **重定向 URL**  
   必须已包含：`https://htma.greatagain.com.cn/api/auth/feishu_callback`（见第二节「安全设置 - 重定向 URL」）。

2. **应用可用性 → 可使用应用的企业**  
   - 打开 [飞书开放平台](https://open.feishu.cn/app) → 进入应用（如 hotmaxxblog）→ **可用性** / **使用范围** / **应用发布**。
   - 在 **「可使用应用的企业」** 中**添加允许其成员发起申请的企业**（如：合作方公司等）。未加入的企业下成员扫码时会提示「没有 xxx 使用权限」或无法选择该应用；加入并保存后，该企业成员**才能完成飞书扫码并进入看板申请流程**。
   - **说明：** 加入某企业仅表示该企业成员可以发起扫码、提交申请；**实际能登录看板的仅为超级管理员在审批页中点击「通过」的个别用户**，同一企业内未获审批的用户无法访问。

3. **权限**  
   应用需已开通 **contact:user.base:readonly**（见第二节「权限」）。

### 7.2 看板侧流程（自动 + 超级管理员）

1. **其他企业用户** 打开 `https://htma.greatagain.com.cn` → 点击「飞书扫码登录」。
2. 飞书授权页选择其所在企业并扫码授权（前提：该企业已在「可使用应用的企业」中）。
3. 回调后看板通过通讯录 API 判断为**企业外用户**，会：
   - 在表 `t_htma_external_access` 中写入或更新一条记录，状态为 `pending`；
   - 通过飞书 webhook **@ 超级管理员**（如余为军），提示有新访问申请；
   - 重定向用户到 **等待审批页** `/pending`（提示「访问申请已提交，等待超级管理员审批」）。
4. **超级管理员** 打开 **审批页**：`https://htma.greatagain.com.cn/approval`（仅超级管理员可访问）。
   - 查看待审批列表（来自 `/api/auth/approvals`）；
   - 点击「通过」或「拒绝」（调用 `/api/auth/approve`）。
5. **审批通过后**，申请者再次打开看板并「飞书扫码登录」，即可正常进入看板。

### 7.3 相关配置

- **超级管理员**：环境变量 `HTMA_SUPER_ADMIN_OPEN_ID`（默认 8db735f2），仅该 open_id 可访问 `/approval`。
- **新申请通知**：环境变量 `FEISHU_WEBHOOK_URL`，用于新申请时 @ 超级管理员。
- **公网地址**：`HTMA_PUBLIC_URL=https://htma.greatagain.com.cn`，用于回调与审批链接。

### 7.4 常见问题

- **扫码提示「没有 xxx 使用权限」**：飞书侧该用户所在企业尚未加入「可使用应用的企业」，需应用管理员在开放平台添加（加入后该企业成员可发起申请，实际可访问的仍仅为经审批的个别用户）。
- **同一企业其他人能直接登录吗？** 不能。看板按**用户**审批，只有被超级管理员在 `/approval` 中点击「通过」的用户才能登录；同一企业内其他人需单独申请并由超级管理员审批。
- **扫码后一直停在「等待审批」**：需超级管理员在 `/approval` 通过该用户；通过后用户需**再次**扫码登录。
- **错误码 20029**：重定向 URL 与飞书控制台不一致，见第四节。
- **`/api/auth/pending_count` 返回 404**：说明当前运行的看板进程未包含该接口（多为部署后未重启）。请**重启看板服务**后再试。前端已做降级：404 时不展示首页管理员提醒条，仅 `/admin` 页会提示「接口未就绪，请重启看板服务」。
