# 经营分析「都没有数据」排查说明

当经营分析页（退货/赠送、品牌、供应商等）显示全为 0 或「暂无数据」时，按下面步骤排查。

---

## 一、看板自检（优先）

1. **打开「经营分析」页**，看顶部 **数据状态条**（子 Tab 下方）：
   - **若显示「未检测到销售数据」**  
     → 表示 MySQL 中 `t_htma_sale` 当前门店下没有记录。  
     → 请到【数据导入】页导入「销售日报」或「销售汇总」Excel，并导入库存。
   - **若显示「数据范围：YYYY-MM-DD ~ YYYY-MM-DD · 销售 N 条」**  
     → 表示库里已有销售数据。若各分析仍为 0，多半是 **所选 KPI 周期与数据日期不重叠**（见下）。

2. **调整 KPI 周期**  
   - 选「近30天」时，只会统计「今天起往前 30 天」内的数据。  
   - 若数据是更早日期（例如只有 2024 年或 2025 年初），请改为「本月」或点「自定义」选包含数据日期的起止日，再点「查询」。

---

## 二、确认数据是否已导入 MySQL

### 方式 1：用接口看数据状态

在浏览器或 curl 访问（把域名换成实际部署地址）：

```
GET https://htma.greatagain.com.cn/api/data_status
```

若返回里 `sale_count` 为 0 或 `min_date`/`max_date` 为空，说明当前环境 MySQL 没有销售数据或未正确导入。

### 方式 2：在服务器上查库

在部署看板的服务器上执行（按实际库名/账号修改）：

```bash
mysql -h 127.0.0.1 -u root -p htma_dashboard -e "
SELECT COUNT(*) AS sale_rows, MIN(data_date) AS min_date, MAX(data_date) AS max_date
FROM t_htma_sale WHERE store_id='沈阳超级仓';
"
```

- `sale_rows` 为 0 → 需要导入销售数据。  
- 有行数但 `min_date`/`max_date` 不在你选的「近30天」内 → 在看板里改周期或自定义日期即可。

---

## 三、导入要求（保证经营分析有数）

1. **销售数据**  
   - 必须导入 **带完整列的「销售日报」或「销售汇总」**（表头含销售日期、销售数量、销售金额、**退货数量、退货金额、赠送数量、赠送金额** 等）。  
   - 若只用简化版 Excel（列不全），`return_qty`/`return_amount`/`gift_qty`/`gift_amount` 会为空，退货/赠送分析会为 0，但周期销售额、品牌等仍可按有数据的列统计。

2. **门店一致**  
   - 导入时 `store_id` 会默认为「沈阳超级仓」。看板与接口都按该门店筛选，确保导入的是同一门店。

3. **库存**  
   - 周转、库存周转等依赖库存快照，需在【数据导入】页导入库存表。

---

## 四、小结

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 数据状态条提示「未检测到销售数据」 | 库里没有销售记录 | 到【数据导入】导入销售 + 库存 |
| 有数据范围但经营分析仍为 0 | 所选周期与数据日期不重叠 | 改为「本月」或自定义日期后再查询 |
| 退货/赠送为 0 其他有数 | Excel 无退货/赠送列或列为空 | 使用完整版销售日报/销售汇总并重新导入 |

按上述顺序先看「经营分析」页的数据状态条，再根据需要调周期或补导数据即可。
