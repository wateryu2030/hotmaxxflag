# 固定链接完整步骤（从零到 https://htma.greatagain.com.cn）

按顺序做下面几步，做完即可用固定地址访问看板，并发给同事长期使用。

---

## 第一步：本机完成 Cloudflare 隧道授权（仅需一次）

1. 打开**终端**，执行：
   ```bash
   cloudflared tunnel login
   ```
2. 会自动打开浏览器；若未打开，把终端里显示的 URL 复制到浏览器打开。
3. 用 **377728157@qq.com** 登录 Cloudflare（若未登录）。
4. 在列表里选中 **greatagain.com.cn**，点击**「授权」**。
5. 若点「授权」没反应：换 Safari 或无痕窗口重试，或允许浏览器弹窗后重试。
6. 成功后终端会显示 “You have successfully logged in” 或类似提示，可关掉浏览器。

---

## 第二步：运行一键脚本（创建隧道 + 启动看板 + 隧道）

在终端执行：

```bash
cd /Volumes/ragflow/hotmaxx/hotmaxxflag
bash scripts/setup_and_run_cloudflare_tunnel.sh
```

脚本会自动：

- 检查是否已登录（第一步未做会提示先做第一步）
- 创建隧道 `htma-dashboard`（若尚未创建）
- 写入 `~/.cloudflared/config.yml`
- 释放 5002 端口并启动看板
- 启动 Cloudflare 隧道

**注意：** 若 Cloudflare DNS 里还没有 htma 的 CNAME，脚本会**打印一条 DNS 记录**（目标为 `xxxxxxxx.cfargotunnel.com`）。需要到 Cloudflare 添加该 CNAME（见第三步）。若你已在 Safari 里添加过 htma 的 CNAME，且目标与脚本输出的隧道 ID 一致，则无需再改。

---

## 第三步：确认 Cloudflare DNS 有一条 htma 的 CNAME

1. 打开 https://dash.cloudflare.com ，登录 **377728157@qq.com**。
2. 进入站点 **greatagain.com.cn** → 左侧 **DNS** → **记录**。
3. 查看是否已有：
   - **类型**：CNAME  
   - **名称**：htma  
   - **目标**：`<隧道ID>.cfargotunnel.com`（与第二步脚本输出的隧道目标一致）  
   - **代理**：已代理（橙色云）
4. 若没有，点 **「添加记录」**，按上面填写；若已有且目标正确，则跳过。

---

## 第四步：保持两个终端不关

- **运行脚本的那个终端**会一直跑着（显示隧道连接状态），不要关掉。
- 关掉后外网链接会断，需要重新执行第二步。

---

## 第五步：访问与分享

- **本机访问**：浏览器打开 http://127.0.0.1:5002  
- **外网 / 同事**：浏览器打开 **https://htma.greatagain.com.cn**

把 **https://htma.greatagain.com.cn** 发给同事，他们即可长期用该固定地址访问看板。

---

## 以后每次要开看板 + 固定链接

1. 打开终端，执行：
   ```bash
   cd /Volumes/ragflow/hotmaxx/hotmaxxflag
   bash scripts/setup_and_run_cloudflare_tunnel.sh
   ```
2. 保持该终端不关，即可用 https://htma.greatagain.com.cn 访问。

---

## 常见问题

- **脚本提示“请先完成 Cloudflare 登录”**  
  先做第一步，在浏览器里完成授权后再运行脚本。

- **https://htma.greatagain.com.cn 打不开**  
  1）确认运行脚本的终端没关；2）确认 Cloudflare DNS 里 htma 的 CNAME 目标与脚本输出的隧道 ID 一致；3）等 1～2 分钟再试。

- **端口 5002 被占用**  
  脚本会自动释放 5002 再启动看板，一般无需手动处理。
