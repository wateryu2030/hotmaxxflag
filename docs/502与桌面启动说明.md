# 502 Bad Gateway 与桌面应用“失效”说明

## 一、502 是什么

通过 **https://htma.greatagain.com.cn**（Cloudflare 隧道）访问时出现 **502 Bad Gateway**，表示：

- Cloudflare 隧道正常，但**本机看板（127.0.0.1:5002）没有在跑**。
- 隧道把请求转到本机 5002，没有进程监听就返回 502。

## 二、常见原因：看板服务退出码 78（外置盘）

- 看板由 **LaunchAgent**（`com.htma.dashboard`）启动，执行 `scripts/start_htma.sh`。
- 当**项目在外置卷**（如 `/Volumes/ragflow/...`）时，launchd 在登录或锁屏后可能**无法访问该卷**（未挂载或权限限制），导致：
  - `start_htma.sh` 里检测不到 `.venv` 或无法读项目 → **exit 78**；
  - 看板进程退出，5002 无人监听 → 外网访问即 502。

所以：**桌面应用“失效” = 双击后虽然执行了安装脚本，但看板服务因 78 没起来，Cloudflare 访问仍是 502。**

## 三、立刻恢复访问（手动启动看板）

在**项目根目录**执行（系统终端或 Cursor 终端均可）：

```bash
cd /Volumes/ragflow/hotmaxx/hotmaxxflag
nohup bash scripts/start_htma.sh >> logs/dashboard.out.log 2>> logs/dashboard.err.log &
```

几秒后访问 http://127.0.0.1:5002 或 https://htma.greatagain.com.cn 应恢复正常。

## 四、桌面启动已做的改进

- **启动好特卖看板.command** 已更新：
  - 先按原逻辑安装/启动 LaunchAgent；
  - **若等待后 5002 仍不通**（例如外置盘导致 78），会**自动用 nohup 在本会话后台启动看板**，再打开浏览器。
- 因此：**即使项目在外置盘，双击桌面图标后一般也能把看板拉起来**，502 会消失；锁屏或重启后若 again 502，再双击一次桌面图标即可。

## 五、长期稳定建议（可选）

若希望**锁屏/重启后看板一直在线、少遇 502**：

1. **把项目拷到本机盘**（如 `~/hotmaxxflag`），再在**新项目根**执行一次：
   ```bash
   bash scripts/install_launchd_htma.sh
   ```
   然后重新运行「创建桌面快捷方式.command」，让桌面图标指向新路径。
2. 这样 LaunchAgent 访问的是本机路径，不再受外置卷挂载影响，看板更容易在后台常驻。

## 六、自检命令

```bash
# 看板是否在跑
lsof -i :5002

# LaunchAgent 状态（第二列为退出码，78 表示未正常启动）
launchctl list | grep com.htma.dashboard

# 看板错误日志
tail -30 /Volumes/ragflow/hotmaxx/hotmaxxflag/logs/dashboard.err.log
```
