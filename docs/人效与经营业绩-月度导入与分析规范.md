# 人效与经营业绩 — 月度导入与分析规范

## 一、目标

- **汇总归类**：将你提供的月度表按**类目**归集，不按个人展示。
- **剔除人员因素**：分析结果只呈现**部门/岗位/业务线/门店等类目**的汇总指标，不出现具体姓名、工号等身份信息。
- **人效分析**：按类目计算人均产值、人均销售额、人均毛利、投入产出比等。
- **经营业绩**：按类目汇总收入、成本、利润、毛利率、净利率、ROI 等。
- **总体核算**：全公司或全门店一层做汇总，与类目明细一致。
- **每月导入**：你每月导入一张表，系统按本规范自动归类、核算并分析。

---

## 二、类目体系（归集维度）

以下类目用于**归集与分组**，分析时只到类目级别，不落到个人。

| 类目编码 | 类目名称     | 说明                     | 表内对应列示例       |
|----------|--------------|--------------------------|----------------------|
| dept     | 部门         | 组织维度                 | 部门、部门人         |
| role     | 岗位/角色    | 岗位或角色               | 岗位、角色           |
| level    | 职级         | 可选，用于职级维度分析   | 职级                 |
| biz_line | 业务线       | 业务条线                 | 业务线               |
| store    | 门店         | 门店/成本中心            | 门店、成本中心       |
| project  | 项目         | 可选                     | 项目                 |

**归集优先级建议**（从细到粗）：  
`门店 > 业务线 > 部门 > 岗位`。  
即：同一张表可先按「门店」汇总，再按「业务线」「部门」「岗位」做交叉或分层分析；具体以你月度表实际列名为准，在导入时做列名映射。

---

## 三、剔除人员因素 — 仅用于归集的字段

以下字段**只用于归集到类目**，不在分析结果中展示、不参与「按人」的排序或名单输出：

- 雇员编号、工号、姓名  
- 性别、学历  
- 入职日期、离职日期、转正日期、合同到期日期  

即：**先按类目聚合，再只输出类目 + 指标**，不输出「谁在哪个类目」。

## 四、指标归类与核算规则

### 4.1 人效类指标（按类目汇总后计算）

| 指标名称       | 含义           | 计算方式（类目维度）                         |
|----------------|----------------|----------------------------------------------|
| 人均产值       | 人均产出       | 总产出 / 类目人数                            |
| 人均销售额     | 人均销售贡献   | 销售额合计 / 类目人数                        |
| 人均毛利       | 人均毛利贡献   | 毛利合计 / 类目人数                          |
| 人均净利       | 人均净利贡献   | 净利合计 / 类目人数                          |
| 人均收入       | 人均创收       | 实际收入合计 / 类目人数                      |
| 人均单产       | 人均单产       | 按你表内定义（如业务量/人）                  |
| 人效比         | 人效比值       | 按你表内定义（产出/人力成本等）               |
| 综合人效       | 综合人效       | 按你表内定义                                 |
| 投入产出比     | 投入产出比     | 产出 / 投入（如经营总产出/总人力成本）       |
| 人均工时       | 人均工时       | 总工时 / 类目人数（用于辅助解读人效）       |
| 人均经营成本   | 人均经营成本   | 经营成本合计 / 类目人数                      |
| 人均销售利润率 | 人均销售利润率 | 按你表内定义                                 |

**人数**：按类目统计**在职人数**（当月有薪资或工时的唯一人数），不按姓名展示。

## 五、总体核算规则

- **公司/全店层**：所有类目汇总为一条「总体」记录。
  - 人效类：全公司总产出 / 全公司人数、总销售额 / 总人数等。
  - 经营业绩类：全公司销售额、收入、成本、毛利、净利、毛利率、净利率、ROI 等。
  - 人力成本类：全公司总人力成本、培训、招聘等。
- **类目层**：按部门、岗位、业务线、门店等分别汇总，再算各人效与经营业绩指标。
- **一致性**：总体 = 各类目数值之和（金额类）；人均类 = 总体金额 / 总人数，或按类目加权平均（视你需求定）。

## 六、月度表导入格式约定

为便于系统**自动识别与归类**，建议月度表满足以下约定（Excel 或 CSV 均可）：

### 6.1 必备列（列名可映射）

以下列名按「可识别」命名，导入时支持列名映射（如「部门」→ dept）。

**类目维度（用于归集，不展示个人）**  
- 部门、岗位、角色、职级、业务线、门店、成本中心、项目（按实际有则提供）

**人员计数用（不展示名单）**  
- 至少有一种可区分「人数」的方式：如同一类目下每人一行，则按类目去重计数；或表内直接有「人数」列。

**人效与经营业绩**  
- 工时：总工时或 1–12 月工时  
- 薪资与成本：当月薪资、提成、绩效、其他补贴、社保、公积金、总人力成本、培训费用、招聘费用等  
- 收入与利润：销售额、实际收入、其他收入、成本、毛利、净利  
- 比率与产出：毛利率、净利率、ROI、人均产值、人均销售额、人均毛利、人效比、综合人效、投入产出比等（若表内已计算则可直接按类目汇总或再算）

### 6.2 可选：列名映射表

你提供当月表后，可在系统内或配置文件中维护「表列名 → 规范指标」的映射，例如：

| 表头列名         | 规范指标/类目 |
|------------------|----------------|
| 部门/部门人      | dept           |
| 岗位             | role           |
| 12月总工时       | total_hours    |
| 合计薪资         | total_salary   |
| 实得收入         | actual_income  |
| 公司实际成本     | company_cost   |
| 费用总额(人力成本) | total_hr_cost  |
| 人均销售额       | sales_per_capita |
| …                | …              |

### 6.3 自动去除表头与无效内容

- **Excel 与附图导入**均会自动过滤，只保留有效数据行：
  - 跳过表头行（岗位列为「岗位」字样、求和项:岗位）、合计/总计/小计/汇总行、空行、纯序号行。
  - 仅导入**组长工资**表（岗位 + 合计薪资、实际薪资、奢品奖金、实得收入、公司实际成本、费用总额）与**组员工资**表（岗位 + 总工时、基本工资、绩效、岗位补贴、合计薪资、奢品、实得收入、公司实际成本）。
- 导入后按**类目**合并计算：组长类目总工资、组员类目总工资、**类目总体工资**（组长+组员合计），并在「查看 1 月分析」中展示。

### 6.4 附图导入（截图/照片 OCR 转录）

- 除 Excel 外，支持**上传附表截图或照片**（png/jpg/jpeg），系统通过 **OCR（Tesseract）** 识别表头与数据行后写入 MySQL，同样会**自动去除表头与无效行**，只取附图内的有效表格内容。
- **环境要求**：服务器需安装 Tesseract 及中文语言包（如 macOS：`brew install tesseract tesseract-lang`）；Python 依赖 `Pillow`、`pytesseract`（见 htma_dashboard/requirements.txt）。
- 图片需包含清晰表头（岗位、合计薪资、费用总额 或 总工时、基本工资 等），识别结果与 Excel 导入写入同一张表，可随后在「查看 1 月分析」中核对类目总体工资。若 OCR 不可用或识别失败，请改用 Excel 导入。

## 七、自主分析流程（每月导入后）

1. **接收月度表**：上传 Excel/CSV。
2. **解析与清洗**：识别类目列、金额列、比率列；统一数值格式；剔除仅用于个人的标识列在**输出**中的展示。
3. **按类目归集**：按部门、岗位、业务线、门店等维度汇总金额与人数（不导出个人名单）。
4. **计算人效**：按类目计算人均产值、人均销售额、人均毛利、投入产出比等（若表内已有则可汇总或复核）。
5. **计算经营业绩**：按类目汇总收入、成本、利润，并算毛利率、净利率、ROI 等。
6. **总体核算**：生成公司/全店合计，并与类目加总核对。
7. **输出**：按类目的分析表 + 总体核算表；可生成简要结论（如人效最高/最低类目、经营业绩达标情况等）。

## 八、与现有看板的关系

- 本规范作为**人效与经营业绩**月度分析的统一约定，可与现有「销售/库存/毛利」等看板并存。
- 后续若在好特卖看板中实现「人效与经营业绩」模块，将按本规范：
  - 增加月度表上传入口；
  - 按类目存储汇总结果（不落个人）；
  - 提供按类目的人效与经营业绩报表及总体核算视图。

## 九、附录：表结构示例（供实现参考）

若入库存储，可按类目+月份存汇总结果，不存个人明细，例如：

- **表名**：`t_htma_eff_performance`（人效与经营业绩汇总）
- **维度**：`report_month`（报表月份）、`category_type`（类目类型：dept/role/biz_line/store）、`category_value`（类目取值，如「市场部」「运营岗」）
- **指标**：人数、总工时、总人力成本、销售额、毛利、净利、人均销售额、人均毛利、投入产出比、毛利率、净利率等（与第四节一致）
- **总体**：`category_type='total'` 且 `category_value='全部'` 表示总体核算一行

以上设计保证：**只做类目级汇总与总体核算，剔除人员因素，便于每月导入后自动分析人效与经营业绩。**
