# 分店商品档案表说明与数据分析字段

## 1. 表与数据源

- **MySQL 表名**：`t_htma_product_master`
- **建表脚本**：`scripts/19_create_product_master_table.sql`
- **数据源**：好特卖导出的「分店商品档案_YYYYMMDD_*.xlsx」，一表对应一次全量导出，可按 `store_id` + `archive_date` 区分多店、多期。

---

## 2. 数据构成概览

档案为**分店维度商品主数据**，每条一行商品（货号唯一），包含状态、价格、品类、供应商、经营属性、服饰属性等。

| 大类 | 字段举例 | 说明 |
|------|----------|------|
| 标识与状态 | 货号、国际条码、品名、商品状态 | 唯一键为 (store_id, sku_code)；状态如「新品」「正常」 |
| 品类 | 类别编码、类别 | 与销售/库存中的品类可做关联分析 |
| 供应商 | 供应商编码、供应商 | 做供应商集中度、品类×供应商分析 |
| 价格 | 零售价、会员价、批发价、划线价、最低售价等 | 多档价格，可做价格带、折扣率分析 |
| 单位与规格 | 单位、规格、产地、进货规格、出货规格 | 规格化与单位统一后可做销量/库存换算 |
| 经营属性 | 商品类型、经销方式、维护库存、分店变价、保质期、损耗率等 | 区分购销/代销、是否管库存、是否生鲜等 |
| 品牌与课组 | 品牌编码、品牌、课组、助记码、商品简称 | 品牌分析、课组结构分析 |
| 人员与日期 | 建档人、建档日期、最后修改人、修改日期、停购日期 | 生命周期、停购商品分析 |
| 服饰扩展 | 性别、上下装、风格、事业部、色系、色深、标准码等 | 服饰品类做属性分析、选品 |

---

## 3. 适合做数据分析的字段（按用途）

### 3.1 商品状态与生命周期

| 字段 | 用途 |
|------|------|
| **product_status**（商品状态） | 区分「新品」「正常」等，做新品占比、状态分布、与销售/库存关联看动销与滞销 |
| **created_at**（建档日期） | 新品时间、商品年龄分布、按上架时间做同期群分析 |
| **updated_at**（修改日期） | 最近维护活跃度、变价/改品频率 |
| **stop_purchase_date**（停购日期） | 停购商品清单、清尾与汰换分析 |

### 3.2 价格与毛利分析

| 字段 | 用途 |
|------|------|
| **retail_price**（零售价） | 价格带分布、与销售表关联算实销单价与折扣 |
| **list_price**（划线价） | 折扣率 = 零售价/划线价 或 实销金额/划线价 |
| **member_price** / member_price_1/2 | 会员价带、会员价与零售价差异 |
| **min_sale_price**（最低售价） | 价格底线、与实销对比防破价 |
| **wholesale_price**（批发价） | 与零售价结合看加价率、毛利空间（需与成本区分） |

与 `t_htma_sale`、`t_htma_profit` 关联后，可做：**品类/品牌价格带 × 销量/毛利**、**新品价格带与动销**。

### 3.3 品类、品牌、课组

| 字段 | 用途 |
|------|------|
| **category_code** / **category_name**（类别编码/类别） | 与销售、库存、毛利表品类关联；品类结构、品类×状态/价格带 |
| **brand_code** / **brand_name**（品牌） | 品牌数量、品牌×品类、品牌×价格带、品牌×动销 |
| **class_group**（课组） | 课组结构、课组×品类/品牌，与业务组织对齐 |

### 3.4 供应商与经营方式

| 字段 | 用途 |
|------|------|
| **supplier_code** / **supplier_name** | 供应商集中度、品类×供应商、供应商×动销/毛利 |
| **distribution_mode**（经销方式） | 购销 vs 代销占比、按经销方式看毛利与库存差异 |
| **product_type**（商品类型） | 普通商品/捆绑商品/代销等，与销量、库存结构结合 |

### 3.5 库存与损耗

| 字段 | 用途 |
|------|------|
| **maintain_stock**（维护库存） | 是否管库存，与库存表、断货/积压分析结合 |
| **shelf_life**（保质期）、**expiry_warning_days**（到期预警天数） | 效期管理、临期品分析（需与库存批次/生产日期结合时再扩展） |
| **loss_rate**（损耗率） | 生鲜/短保品损耗预估、与实销/报损对比 |
| **is_fresh**（生鲜商品） | 区分生鲜与非生鲜，做不同分析维度 |

### 3.6 服饰与选品（有数据时）

| 字段 | 用途 |
|------|------|
| **gender**、**clothing_type**（上下装）、**style**（风格） | 性别×品类、上下装×销量、风格分布 |
| **division**（事业部）、**color_family**（色系）、**color_depth**（色深） | 事业部结构、色系/色深与销量、库存搭配 |

### 3.7 关联分析建议

- **与销售**：`t_htma_sale.sku_code` ↔ `t_htma_product_master.sku_code`（同 store_id），用档案的**零售价/划线价/品类/品牌**做价格带、折扣率、品类/品牌表现。
- **与库存**：`t_htma_stock.sku_code` ↔ `t_htma_product_master.sku_code`，用档案的**商品类型、维护库存、保质期**做库存健康度、效期分析。
- **与毛利**：通过 sku → 品类/品牌 关联到毛利表，做**品类/品牌 × 档案属性（价格带、经销方式）**的毛利结构分析。

## 4. 使用注意

1. **货号**：档案中货号可能含前导零或位数不一（如 0、0000000000、A11M341904F），关联销售/库存时需统一 trim 与长度处理，或保留原样做精确匹配。
2. **唯一性**：同一分店同一货号在单次导出中应唯一；若同一货号多行（如不同规格/条码），当前表以 (store_id, sku_code) 唯一，后续可按需增加「规格/条码」维度。
3. **档案日期**：`archive_date` 建议填文件名中的日期（如 20260306），便于按导出批次做增量或全量对比。
4. **导入与 OpenClaw**：
   - **建表**：手工执行 `mysql ... htma_dashboard < scripts/19_create_product_master_table.sql`，或一键 `bash scripts/run_product_master_create_and_import.sh`（仅建表）。
   - **Web 导入**：登录后打开「数据导入」页，在「分店商品档案导入」区块上传 Excel，或使用同权限的「商品档案」链接查看分析。
   - **脚本/OpenClaw**：`python scripts/openclaw_product_master_import_from_downloads.py` 从下载目录找 `分店商品档案_*.xlsx` 并导入；`run_product_master_create_and_import.sh --import` 可建表后顺带导入。
   - 权限与「数据导入」「人力成本」同级：需登录且具备 `product_master` 模块权限（超级管理员默认有；或配置 `HTMA_PRODUCT_MASTER_ALLOWED_FEISHU_OPEN_IDS`）。
