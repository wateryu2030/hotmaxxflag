# 自定义时间起点 · 实现模式

本文档约定「自定义时间起点」的通用实现模式，便于 KPI 与人力成本等模块复用。

## 一、模式要点

1. **前端**
   - **周期 Tab**：提供「自定义」与固定周期（今日/本周/本月/近30天）。
   - **起止日期区**：当选择「自定义」时展示「起止日期：`[起始日期]` 至 `[结束日期]` + 应用」。
   - **日期范围来源**：页面加载或切到「自定义」时请求 `/api/date_range` 得到 `min_date`、`max_date`，用于设置 `<input type="date">` 的 `min`/`max`，并去掉 `readonly`/`disabled`。
   - **应用**：用户选好起止日期后点「应用」，将 `currentStartDate`/`currentEndDate` 带入后续数据请求（如 `start_date`、`end_date` 查询参数）。

2. **后端**
   - **日期范围接口**：`GET /api/date_range` 返回 `{ "min_date": "YYYY-MM-DD", "max_date": "YYYY-MM-DD" }`，无数据或异常时返回近一年范围，且不要求登录。
   - **业务接口**：支持 `start_date`、`end_date` 查询参数；当两者均提供时按自定义区间筛选，否则按 `period`（今日/本周/本月/近30天）计算区间。

3. **人力成本扩展（规划）**
   - **当前**：人力成本按「报表月份」（YYYY-MM）展示与导入，仅完成导入与分月展示。
   - **后续**：人力成本页将支持「按自定义起止日期分解」：已知某月总人力成本，在用户选择起止日期后，按日或按在岗天数等规则分解到该区间计算展示。实现时复用上述前端模式（起止日期 + 应用）及后端 `start_date`/`end_date` 参数约定。

## 二、KPI 已实现位置（参考）

- 前端：`htma_dashboard/static/index.html`  
  - 周期 Tab：`.period-tabs`、`#tabCustom`、`#dateRangeInline`、`#startDate`、`#endDate`、`#btnApplyCustomRange`  
  - 逻辑：`loadDateRange()`、`applyFilter()`、`updatePeriodTabActive()`、周期 Tab 的 click 处理
- 后端：`htma_dashboard/app.py`  
  - `GET /api/date_range`  
  - `_date_condition(period, start_date, end_date)`、各 KPI/品类/明细接口的 `start_date`/`end_date` 参数

## 三、人力成本当前与后续

- **当前**：人力成本 Tab 仅「报表月份」输入 + 分月展示；导入完成即可用。
- **后续**：增加「自定义起止日期」时，前端可复用 KPI 的 date-range-inline 组件与交互，后端 labor 接口增加 `start_date`/`end_date`，在区间内对当月人力成本做分解计算（规则待定：按日占比或按在岗等）。
