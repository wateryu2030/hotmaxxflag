# 定时比价任务说明（OpenClaw 自动化）

## 1. 功能概览

- **比价可用性检查**（每日 8:00）：运行 `run_selfserve_price_compare_debug.sh` 自检，结果写入 `logs/price_compare_healthcheck.log`，用于 OpenClaw 自动化**检查平台比价功能是否可用**。安装：`bash scripts/install_launchd_price_compare_healthcheck.sh`。
- **实际比价任务**（每周一 2:00）：`scripts/batch_price_compare.py` 从商品档案与近 30 天销售中取大额商品，调用百度优选 Skill 比价并写入 `t_price_compare`。安装：`bash scripts/install_launchd_price_compare.sh`。

## 2. 前置条件

- 已执行 `scripts/23_create_t_price_compare.sql` 建表。
- 已安装百度优选 Skill：`clawhub install baidu-ecommerce-skill`（或项目要求的等价方式）。
- 项目具备 `.venv` 与 `.env`（含 `MYSQL_*`、可选 `HTMA_STORE_ID`、`PRICE_COMPARE_TOP_N`、`PRICE_COMPARE_MIN_PRICE`、`PRICE_COMPARE_DELAY`）。

## 3. 安装定时任务（launchd）

在项目根目录执行：

```bash
bash scripts/install_launchd_price_compare.sh
```

将安装 `~/Library/LaunchAgents/com.htma.price_compare.plist`，每周一 02:00 运行 `scripts/run_price_compare_cron.sh`，该脚本会调用 `batch_price_compare.py` 并写日志。

## 4. 手动执行与日志

- **手动跑一次**：
  ```bash
  bash scripts/run_price_compare_cron.sh
  ```
- **日志位置**：
  - `logs/price_compare_cron.log`：脚本内追加的比价过程与结果；
  - `logs/price_compare_cron.out.log` / `logs/price_compare_cron.err.log`：launchd 的 stdout/stderr。

## 5. 比价可用性检查（每日自检）

- **安装**：`bash scripts/install_launchd_price_compare_healthcheck.sh`
- **Label**：`com.htma.price_compare_healthcheck`，每日 8:00 执行。
- **日志**：`logs/price_compare_healthcheck.log`（自检输出）、`.out.log` / `.err.log`。
- **手动跑一次**：`bash scripts/run_price_compare_healthcheck.sh`
- 自检通过时日志会显示「全部通过，看板货盘比价应能使用百度 Skill」；若网关未暴露比价 tool 或 npm 版 ClawHub 无 `run`，会记录失败步骤，便于排查。详见 [百度Skill比价环境说明.md](./百度Skill比价环境说明.md)。

## 6. 常用命令

- 查看是否已加载：
  - 比价可用性检查：`launchctl list | grep com.htma.price_compare_healthcheck`
  - 实际比价任务：`launchctl list | grep com.htma.price_compare`
- 停止/再次启用：`launchctl unload` / `launchctl load` 对应 plist（见各安装脚本输出）。

## 7. 与看板、防睡眠的关系

- 定时比价**不依赖**看板服务是否启动，只要本机在周一 02:00 未关机即可执行。
- 若希望锁屏/息屏后本机不睡眠、定时任务稳定触发，可先安装防睡眠与看板：`bash scripts/install_launchd_htma.sh`（见 [人力成本API与部署说明.md](./人力成本API与部署说明.md) 第 4 节）。

## 8. 可选：crontab 方式

若不想用 launchd，可用 crontab 每周一 02:00 执行：

```bash
0 2 * * 1 /bin/bash /path/to/项目根/scripts/run_price_compare_cron.sh
```

将 `/path/to/项目根` 替换为实际路径；确保 cron 环境能访问 `.env` 与 `.venv`（或在该脚本内已 source `.env` 并使用绝对路径的 python）。
