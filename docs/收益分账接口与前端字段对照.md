# 收益分账：接口 JSON 与前端字段对照

## 计算接口 POST /api/profit_share/calculate

**返回结构（扁平，与前端 `renderCalcResult(d)` 一致）：**

| 后端字段 | 前端使用 | 说明 |
|----------|----------|------|
| total_sales | d.total_sales | 参与计算的总销售额 |
| total_cost | d.total_cost | 销售成本（= 销售额 - 总毛利） |
| total_profit | d.total_profit | 总毛利（分账基数，仅对此按比例分） |
| merchant_settle_amount | d.merchant_settle_amount | 加盟商结算 = 总毛利×加盟商% |
| platform_amount | d.platform_amount | 平台留存 = 总毛利×平台% |
| share_rate_merchant | d.share_rate_merchant | 加盟商比例（如 75） |
| share_rate_platform | d.share_rate_platform | 平台比例（如 25） |
| share_rate_used | d.share_rate_used | 当前规则使用的加盟商比例 |
| rule_name | d.rule_name | 规则名称（如「前2年」） |
| period_label | d.period_label | 计算周期文案 |
| calc_month | d.calc_month | 月份或周期 |
| total_sales_all | d.total_sales_all | 全口径销售额 |
| total_profit_all | d.total_profit_all | 全口径毛利 |
| excluded_sales | d.excluded_sales | 排除项销售额 |
| excluded_profit | d.excluded_profit | 排除项毛利 |
| exclusion_summary | d.exclusion_summary | 排除说明文案 |
| formula_note | d.formula_note | 公式说明 |
| details | d.details | 按品类明细 [{ category_large_name, category_mid_name, sales, profit }] |

**计算逻辑（已统一）：**

- 加盟商结算 = `round(total_profit * (rate_merchant/100), 2)`，且 ≤ total_profit  
- 平台留存 = `round(total_profit * (rate_platform/100), 2)`，且 ≤ total_profit  
- 严禁使用销售额参与分账计算。

## 历史详情接口 GET /api/profit_share/result/:id

返回 `{ result: row, details: [...] }`。前端将 `row` 与 `details` 拼成与上面相同的扁平结构传给 `renderCalcResult(combined)`。

详情接口会对 `merchant_settle_amount`、`platform_amount` 按 **总毛利与比例** 重新计算后写入 `row`，避免历史错误数据导致展示错误。

## 前端表格展示

- 一、数据概览：全口径 / 参与计算 / 排除项（销售额、毛利额）
- 二、分账结果：总销售额 → 销售成本 → 总毛利 → 分成比例 → 加盟商结算金额 → 平台留存金额

字段名前后端一致，无需前端再做换算。
