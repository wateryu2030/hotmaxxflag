# 现有数据还能做哪些经营性分析

本文档基于当前数据资产与已实现能力，梳理：**已实现的经营分析**、**仅用现有表即可新增的分析**，以及**可选的 API/看板建议**。不涉及需要补充订单号/客流等新数据的指标（连带率、提袋率等见 [现有数据情况与经营指标分析可行性.md](./现有数据情况与经营指标分析可行性.md)）。

---

## 一、数据资产与字段确认

### 1.1 核心表与粒度

| 表名 | 粒度 | 说明 |
|------|------|------|
| **t_htma_sale** | 日期 × SKU | 每日每商品销售汇总，**无订单号/小票号** |
| **t_htma_profit** | 日期 × 品类 | 每日每品类销售额与毛利 |
| **t_htma_stock** | 日期 × SKU | 库存快照 |
| **t_htma_category_profit** | 品类 × 周期 | 周期内品类汇总，sale_count = 有销售天数 |
| **t_htma_tax_burden** | 编码 | 税率负担表（编码→税率等） |
| **t_htma_category** | 编码 | 品类编码与名称映射（用于税率计算等） |

### 1.2 销售表 t_htma_sale 可用字段（已确认有数据）

- **基础**：data_date, sku_code, category, sale_qty, sale_amount, sale_cost, gross_profit  
- **退货/赠送**：return_qty, return_amount, gift_qty, gift_amount（汇总有值：退货约 12.86 万、赠送约 1.17 万）  
- **品牌/供应商**：brand_name, supplier_name（约 4.5 万条有值）  
- **品类层级**：category_large, category_mid, category_small  
- **价格与成本**：sale_price, avg_sale_price, cost_amount  
- **商品信息**：product_name, spec, barcode 等

## 二、已实现的经营分析（API + 看板 + 报告）

以下均为当前系统已支持的能力，可直接使用。

### 2.1 KPI 与汇总

| 能力 | 接口/入口 | 说明 |
|------|-----------|------|

### 2.2 趋势与结构

| 能力 | 接口/入口 | 说明 |
|------|-----------|------|
| 日/周/月趋势 | `/api/daily_trend`、`/api/sales_trend`、`/api/trend_analysis` | 销售额/销量趋势、同比环比 |
| 品类占比 | `/api/category_pie` | 品类销售占比（饼图） |
| 周几销售 | `/api/dow_sales` | 星期几销售分布 |

### 2.3 库存与预警

| 能力 | 接口/入口 | 说明 |
|------|-----------|------|
| 库存预警 | `/api/inv_alert`、`/api/inv_alert_list`、`/api/inv_alert_by_category` | 低库存、断货风险、按品类 |
| 数据状态 | `/api/data_status` | 数据新鲜度、最后更新日期 |

### 2.4 税率与税务

| 能力 | 接口/入口 | 说明 |
|------|-----------|------|
| 税率计算 | `/api/tax_burden_summary`、税率计算页签 | 按税率汇总销售额/不含税/税额，按税率下品类明细 |
| 税率导出 | `/api/tax_burden_export?format=csv\|pdf` | Excel(CSV)/PDF 导出 |

### 2.5 智能洞察与营销报告

| 能力 | 接口/入口 | 说明 |
|------|-----------|------|
| 智能洞察 | `/api/insights`、analytics.build_insights | 低毛利品类、二八集中度、低库存预警、负毛利、整体毛利率、动销与周转提示 |

### 2.6 品类排行

| 能力 | 接口/入口 | 说明 |
|------|-----------|------|
| 大类排行 | `/api/category_rank_by_large` | 按大类销售排序 |
| 品类排行 | `/api/category_rank`、`/api/category_rank_detail` | 品类销售与排名 |

### 2.7 其他

- 商品/品类主数据：`/api/products`、`/api/categories`  
- 销售明细：`/api/sale_detail`  
- 导出：`/api/export`

## 三、仅用现有表即可新增的经营性分析

以下均**不依赖新表或新数据源**，仅基于现有 t_htma_sale / t_htma_profit / t_htma_stock 等即可实现（部分需在导入时已填充 return_*/gift_*、brand_name、supplier_name、sale_price 等，当前库已具备）。

### 3.1 退货与赠送分析

- **退货占比**：按日/周/月：退货金额 ÷ 销售额、退货件数 ÷ 销售件数。  
- **赠送占比**：赠送金额/件数占销售的比例，按品类或大类。  
- **高退货品类/SKU**：退货率高于均值的品类或 SKU 列表，便于排查质量或客诉。  
- **用途**：控制损耗、优化供应商或品控、区分正常退货与异常退货。

**实现要点**：  
- 汇总：`SUM(return_amount)/SUM(sale_amount)`、`SUM(return_qty)/SUM(sale_qty)`，按 data_date 或 category/category_large 分组。  
- 可新增接口如 `/api/return_gift_summary`，看板增加「退货与赠送」卡片或小页。

---

### 3.2 品牌与供应商贡献分析

- **品牌**：按 brand_name 汇总销售额、毛利、销量；品牌销售占比、毛利贡献 Top N；品牌毛利率排名。  
- **供应商**：按 supplier_name 汇总销售额、毛利；供应商集中度（前 N 家占比）；供应商毛利率分布。  
- **用途**：采购谈判、汰换低效品牌/供应商、优化品类结构。

**实现要点**：  
- 从 t_htma_sale 按 brand_name / supplier_name 分组，聚合 sale_amount、gross_profit、sale_qty。  
- 可新增 `/api/brand_summary`、`/api/supplier_summary`，看板增加「品牌分析」「供应商分析」模块。

---

### 3.3 价格带与件单价分析

- **件单价分布**：按 (sale_amount/sale_qty) 或 sale_price 分段（如 0–10、10–30、30–50、50+ 元），看各价格带销售额/销量占比。  
- **品类件单价**：各品类平均件单价、中位数，识别高价带品类与低价带引流品类。  
- **用途**：定价策略、促销选品、价格带与毛利结构优化。

**实现要点**：  
- 使用 sale_price 或 `sale_amount/sale_qty` 做 CASE WHEN 分段，按段聚合。  
- 可新增 `/api/price_band_summary`，看板增加「价格带分布」图。

---

### 3.4 SKU 周转与 ABC 分析

- **周转率/周转天数**：  
  - 按 SKU：近 N 天销量、当前库存（最新 stock 快照），周转天数 = 当前库存 ÷ 日均销量（或类似口径）。  
  - 按品类：品类总库存 ÷ 品类日均销量。  
- **ABC 分类**：按销售额或毛利对 SKU 排序，累计占比 A(80%)/B(15%)/C(5%)，或自定义比例；可再结合周转天数标「快/中/慢」。  
- **用途**：补货优先级、陈列与采购资源分配、滞销清库。

**实现要点**：  
- 销售汇总 JOIN 最新库存，算日均销量与周转天数。  
- 累计占比用窗口函数或应用层排序均可。  
- 可新增 `/api/sku_turnover`、`/api/sku_abc`，看板增加「周转与 ABC」页或区块。

---

### 3.5 负毛利与成本异常明细

- **负毛利明细表**：按 SKU 列出销售额>0 但毛利<0 的明细（含销量、销售额、成本、毛利、毛利率），支持按品类/大类筛选和导出。  
- **成本异常**：成本为 0 或明显偏离同品类均值的 SKU（需定义阈值），便于排查录入错误。  
- **用途**：与现有「负毛利」洞察和营销报告中的负毛利清单形成互补，做专项报表与整改跟踪。

**实现要点**：  
- 基于 t_htma_sale 筛选 gross_profit < 0 且 sale_amount > 0，按 sku_code 聚合；可关联 t_htma_stock 取品名/规格。  
- 可新增 `/api/negative_margin_detail`，支持 CSV 导出；看板可增加「负毛利明细」表格页。

---

### 3.6 品类结构随时间变化

- **品类占比趋势**：按周/月计算各品类（或大类）销售额占比，看占比随时间上升/下降。  
- **新兴/衰退品类**：占比连续 N 期上升 vs 连续下降的品类。  
- **用途**：品类规划、季节性备货、资源倾斜。

**实现要点**：  
- 按 data_date  truncate 到周/月，按 category 或 category_large 汇总 sale_amount，再算占比。  
- 可复用或扩展 `/api/trend_analysis`，或新增 `/api/category_structure_trend`。

---

### 3.7 库存金额与周转（门店/整体）

- **库存金额趋势**：按日/周取库存总金额（t_htma_stock.stock_amount），观察资金占用变化。  
- **库存周转天数（整体）**：期末库存金额 ÷ (近 N 天销售成本或销售额/对应天数)×天数，得到「周转天数」口径。  
- **用途**：资金占用、库存健康度、与销售规模的匹配度。

**实现要点**：  
- 库存金额：按 data_date 汇总 t_htma_stock；销售成本可从 t_htma_sale 或 t_htma_profit 取。  
- 可新增 `/api/inventory_turnover_summary`，与现有库存预警并列展示。

---

### 3.8 数据质量与一致性

- **成本/售价缺失**：sale_price 或 cost 为 NULL/0 的 SKU 数量及涉及金额，按日或按品类。  
- **品类一致性**：同一 sku_code 在不同日期的 category 是否一致，异常清单。  
- **用途**：数据清洗、导入校验、报表可信度。

**实现要点**：  
- 简单 COUNT/SUM 与 GROUP BY；品类一致性需按 sku_code 分组检查 category 去重后是否唯一。  
- 可作为 `/api/data_status` 的扩展或单独 `/api/data_quality`。

## 四、可选的 API / 看板模块建议（不强制实现）

| 分析项 | 建议接口 | 建议看板位置 |

## 五、小结

- **已有能力**：KPI、趋势、品类占比、周几、库存预警、税率计算与导出、insights、营销报告、品类排行等已覆盖主要经营看数与报告需求。  
- **可直接扩展**：在不新增数据源的前提下，现有表已支持**退货/赠送、品牌/供应商、价格带、SKU 周转与 ABC、负毛利明细、品类结构趋势、库存周转、数据质量**等分析；实现方式以新增或扩展 API + 看板模块为主。  
- **仍不可做**：连带率（件数/订单笔数）、提袋率（成交/进店）等依赖订单号或客流的数据，需业务补充后再做（见 [现有数据情况与经营指标分析可行性.md](./现有数据情况与经营指标分析可行性.md)）。

若需要优先实现其中某几项，可从「退货与赠送」「品牌/供应商贡献」「负毛利明细」或「SKU 周转/ABC」任选，再按上表落接口与前端即可。
