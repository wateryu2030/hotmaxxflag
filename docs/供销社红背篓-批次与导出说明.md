# 供销社「红背篓」选品：批次效期（方案 A）与导出

## 1. 数据库

首次使用前执行建表：

```bash
mysql -h 127.0.0.1 -u root -p htma_dashboard < scripts/25_create_sku_batch_table.sql
```

表 **`t_htma_sku_batch`**：按 SKU + 批次记录 **`expiry_date`（到期日）** 与 **`qty`（件数）**。  
选品逻辑对同一 SKU 聚合 **`MIN(expiry_date)`** 作为「最早到期日」，并汇总 `SUM(qty)` 为批次表合计（可与总库存对照，不要求与 `t_htma_stock` 完全一致）。

## 2. 页面与权限

- 访问：**`/hongbeilou`**（需登录，且与「数据导入」相同：拥有 **import** 模块权限）。
- 操作：选品类、填 **加价率**，预览；**导出 CSV**、**导出 PDF**、**导出 PDF（水印）**。预览与导出列一致（见下）；数据库仍保留全量字段备查。
- **分组展示**：勾选「按大类→中类→小类级联」时，页面用 **多选级联** 展示——先勾选大类后仅显示这些大类下的中类；勾选中类后仅显示对应小类；再勾选小类后，底部表格仅展示三者组合均命中的 SKU（数据与 **`GET .../preview`** 返回的 `rows` 一致，无额外接口）。

### API：选品逻辑说明（不拉全表）

- **`GET /api/channel/hongbeilou/logic`**：参数含 `category_*_code`、`share_ratio`、`min_stock`、`exclude_expired`（及可选 `markup`）。返回 `rules`、`column_glossary`、`applied_filters`、环境探测等。
- **`GET /api/channel/hongbeilou/preview`**：同上 + 返回简化 **`rows`**（与 CSV/PDF 列一致），`logic.markup_ratio_effective` 为解析后的加价率。
- **`GET /api/channel/hongbeilou/export`**：下载 CSV（UTF-8 BOM），首列为 **序号**，其后为简化列（无大类），含 **供货价**；Query 参数 **`markup`** = 加价率。
- **`POST /api/channel/hongbeilou/export`**：`Content-Type: application/json`，Body 与 GET 相同筛选字段，另加 **`sku_codes`**（字符串数组）时**仅导出**这些 SKU（与当前筛选 SQL 结果求交）。页面「仅导出已勾选」使用该方式。
- **`GET /api/channel/hongbeilou/export_pdf`**：下载 PDF（首列序号，无大类列；库存/近30天销量/供货价居中）；**`&watermark=1`** 时在**页面内容之上**叠斜向水印 **「宝赞商业＠振鸿」**；正文仅保留面向接收方的 **「本选品单于YYYY年MM月DD日 HH:MM生成，不代表其他时间有效」**（不含供货价计算公式说明）。
- **`POST /api/channel/hongbeilou/export_pdf`**：同上，JSON 中可带 **`sku_codes`**、**`watermark`: true**。
- **依赖**：PDF 需 **`reportlab`**（见 `htma_dashboard/requirements.txt`）。

## 3. 批次数据维护

可通过 SQL 插入，或调用 API（需登录 + import 权限）：

`POST /api/channel/hongbeilou/batch`  
`Content-Type: application/json`

```json
{
  "rows": [
    {
      "sku_code": "123456",
      "expiry_date": "2026-09-01",
      "qty": 48,
      "batch_no": "LOT-001",
      "production_date": "2026-03-01"
    }
  ]
}
```

## 4. 导出字段说明（CSV / PDF）

**序号**（自动生成）+ 品名、规格、单位、品牌、中类、小类、库存数量、近30天销量、**供货价**（**不含大类列**）。

- **供货价** = **我方参考价** × (1 + **加价率**)。参考价取数顺序与后台规则一致（门店售价 → 库存均价 → 商品表单价 → 档案零售价 → 档案配送价）；无参考价则供货价为空。
- **加价率 `markup`**：**不得低于 10%**（至少填 **0.1** 或 **10**）；未传或留空时服务端按 **15%** 处理。填 **0.15** 或 **15** 表示 15%；大于 1 的数字按「百分数÷100」解析。

（批次效期、比价等仍在库内与后台查询中保留，导出文件为对外简化版。）
