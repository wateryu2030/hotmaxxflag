# 固定外网链接：发给同事长期访问（地址不变）

快速隧道（trycloudflare.com）每次启动都会变链接，不适合发给同事长期用。下面两种方式可以得到**固定地址**，发一次即可。

---

## 方案一：Cloudflare 正式隧道（推荐，固定 https 域名）

**结果**：得到一个固定地址，例如 `https://htma.你的域名.com`，发一次给同事，以后一直用这个链接。  
**前提**：有一个域名（可买便宜域名或使用免费子域名），并注册 Cloudflare（免费）。

### 1. 准备域名

- 若还没有域名：可购买便宜域名（如阿里云、腾讯云、Namecheap 等，约 1–10 元/年），或使用能提供“自己域名”的免费服务。
- 在 Cloudflare 添加该域名：登录 [dash.cloudflare.com](https://dash.cloudflare.com) → 添加站点 → 按提示把域名的 DNS 解析改为 Cloudflare 提供的 NS（或保持在其他 DNS 服务商，用 Cloudflare 的 CNAME 方式也可，以 Cloudflare 文档为准）。

### 2. 本机登录 cloudflared

在终端执行（会打开浏览器，用 Cloudflare 账号登录）：

```bash
cloudflared tunnel login
```

按提示选择你的域名，完成授权。

### 3. 创建隧道并记下隧道名称

```bash
cloudflared tunnel create htma-dashboard
```

终端会显示隧道 ID（一串 UUID），并说明凭证已保存。隧道名称这里用 `htma-dashboard`，后面会用到。

### 4. 在 Cloudflare 里为隧道添加 DNS 记录

在 Cloudflare 控制台：**你的域名** → **DNS** → **记录** → 添加：

- **类型**：CNAME  
- **名称**：`htma`（或任意子域名，例如 `dashboard`，则最终地址为 `https://dashboard.你的域名.com`）  
- **目标**：`<隧道ID>.cfargotunnel.com`（把 `<隧道ID>` 换成上一步看到的 UUID，例如 `abcd1234-5678-90ab-cdef-1234567890ab.cfargotunnel.com`）  
- **代理状态**：已代理（橙色云朵）

保存后，你的固定地址就是：**https://htma.你的域名.com**（或你填的名称）。

### 5. 写隧道配置文件

在项目里已准备好示例配置路径，你只需把**隧道名称**和**主机名**改成自己的。创建或编辑文件（例如 `~/.cloudflared/config.yml` 或项目下的 `scripts/cloudflared-htma.yml`）：

```yaml
tunnel: htma-dashboard
credentials-file: /Users/你的用户名/.cloudflared/<隧道ID>.json

ingress:
  - hostname: htma.你的域名.com
    service: http://127.0.0.1:5002
  - service: http_status:404
```

- `tunnel`：与第 3 步创建的隧道名称一致。  
- `credentials-file`：第 3 步创建隧道后生成的 json 文件路径，在 `~/.cloudflared/` 下，文件名是隧道 ID。  
- `hostname`：和第 4 步在 DNS 里填的子域名一致（如 `htma.你的域名.com`）。  
- `service`：本机看板端口，保持 `http://127.0.0.1:5002` 即可。

### 6. 启动看板 + 固定隧道（每次开机或需要时）

先启动看板（本机 5002），再跑隧道：

```bash
# 终端 1：启动看板
cd /Volumes/ragflow/hotmaxx/hotmaxxflag
bash 启动好特卖看板.command
```

```bash
# 终端 2：启动固定隧道（用你的配置文件路径）
cloudflared tunnel --config ~/.cloudflared/config.yml run htma-dashboard
```

之后**固定外网链接**就是你在 DNS 里配的：**https://htma.你的域名.com**。  
把这个链接发给同事，他们即可长期用同一地址访问；你只要保持看板 + 上面这条 `cloudflared tunnel run` 在运行即可。

### 7. 可选：开机自启隧道

若希望重启电脑后隧道也自动起来，可把 `cloudflared tunnel run` 配成系统服务（macOS 用 launchd），这样固定链接会一直有效。需要的话可再写一份 launchd 配置示例。

## 方案二：路由器端口转发 + DDNS（固定 IP 或固定域名）

**结果**：固定地址，例如 `http://你的名字.duckdns.org:5002`，发一次给同事长期用。  
**前提**：本机在路由器后面、路由器有公网 IP（或运营商已分配公网 IP），且你能登录路由器做端口转发。

### 1. 注册 DDNS（免费固定域名）

- 打开 [DuckDNS](https://www.duckdns.org/) 或 [No-IP](https://www.noip.com/) 等，注册并创建一个子域名，例如 `htma`，得到：`htma.duckdns.org`。

### 2. 在路由器上做端口转发

- 登录路由器管理页（如 192.168.1.1）。  
- 找到「端口转发 / 虚拟服务器」：  
  - 外部端口：5002（或 80，若用 80 则访问时不用写端口）。  
  - 内部 IP：运行看板那台 Mac 的局域网 IP（如 192.168.1.100）。  
  - 内部端口：5002。  
- 保存。

### 3. 在路由器或本机配置 DDNS

- 在路由器「DDNS」里填写 DuckDNS/No-IP 的账号和域名，让路由器定期把当前公网 IP 更新到 `htma.duckdns.org`。  
- 或在本机用 DuckDNS 的更新脚本/客户端，定期更新 IP。

### 4. 固定链接

- 若转发的是 5002：**http://htma.duckdns.org:5002**  
- 若转发的是 80：**http://htma.duckdns.org**  

把这个地址发给同事即可长期使用。注意：这是 **http**，若需要 https，可在本机或路由器上再配 Nginx 反代 + 证书，或使用方案一（Cloudflare 自带 https）。

---

## 对比与建议

| 方式           | 固定链接示例                    | 需要条件                     | 难度 |
|----------------|----------------------------------|------------------------------|------|
| Cloudflare 隧道 | https://htma.你的域名.com       | 域名 + Cloudflare 账号       | 中   |
| 端口转发+DDNS  | http://htma.duckdns.org:5002    | 公网 IP + 路由器配置         | 中   |
| 快速隧道（当前）| https://xxx.trycloudflare.com   | 无                           | 简单，但链接每次变 |

若希望**发一次链接给同事、长期不变**，优先用**方案一（Cloudflare 正式隧道）**，链接是 https 且稳定。  
方案二适合不想买域名、但有公网 IP 且能动路由器的情况。

## 发给同事时怎么说

- **方案一**：  
  “看板外网地址是：**https://htma.你的域名.com** ，收藏这个链接即可，以后都打开这个。”  
- **方案二**：  
  “看板外网地址是：**http://htma.duckdns.org:5002** ，收藏即可长期用。”

你这边只要保证：看板已启动，且隧道（方案一）或路由器+DDNS（方案二）正常，同事即可随时用该固定链接实时查看。
