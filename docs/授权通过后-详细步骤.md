# 授权通过后 — 详细步骤（到 https://htma.greatagain.com.cn）

授权已通过（greatagain.com.cn 为 Active），按下面顺序做即可。

---

## 第一步：运行一键脚本（创建隧道 + 启动看板 + 隧道）

1. 打开**终端**。
2. 执行：
   ```bash
   cd /Volumes/ragflow/hotmaxx/hotmaxxflag
   bash scripts/setup_and_run_cloudflare_tunnel.sh
   ```
3. 脚本会自动：
   - 创建隧道 `htma-dashboard`（若尚未创建）
   - 写入 `~/.cloudflared/config.yml`
   - 释放 5002 端口并启动看板
   - 启动 Cloudflare 隧道
4. 终端里会输出 **隧道 ID** 和一行 **目标**，形如：  
   `目标: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx.cfargotunnel.com`  
   **不要关这个终端窗口**，关掉外网链接会断。

---

## 第二步：确认 Cloudflare DNS 里 htma 指向该隧道

1. 打开 https://dash.cloudflare.com ，登录 **377728157@qq.com**。
2. 进入 **greatagain.com.cn** → 左侧 **DNS** → **记录**。
3. 查看是否有一条：
   - **类型**：CNAME  
   - **名称**：htma  
   - **内容/目标**：`<第一步脚本输出的隧道ID>.cfargotunnel.com`  
   - **代理**：已代理（橙色云）
4. **若没有**：点「添加记录」，类型选 CNAME，名称填 `htma`，目标填第一步脚本里打印的那串 `xxxxxxxx.cfargotunnel.com`，代理开启，保存。
5. **若已有但目标不一致**：编辑该记录，把目标改成脚本输出的隧道 ID 对应的 `xxxxxxxx.cfargotunnel.com`，保存。

---

## 第三步：访问与分享

- **本机**：浏览器打开 **http://127.0.0.1:5002**
- **外网 / 同事**：浏览器打开 **https://htma.greatagain.com.cn**

把 **https://htma.greatagain.com.cn** 发给同事即可长期使用。  
若刚改过 DNS，可等 1～2 分钟再访问。

---

## 本机锁屏后仍可访问（推荐）

若希望**关掉终端、锁屏或息屏后**仍能通过外网链接打开看板，请用 **launchd 后台服务** 方式：

1. 在项目根目录创建 `.tunnel-token`，内容为一行：Cloudflare 隧道 Token（从 one.dash.cloudflare.com 复制）。
2. 执行：
   ```bash
   cd /Volumes/ragflow/hotmaxx/hotmaxxflag
   bash scripts/install_launchd_htma.sh
   ```
3. 安装后，看板与隧道以后台服务运行，**本机锁屏、关终端后仍可访问** https://htma.greatagain.com.cn。隧道脚本会阻止本机进入睡眠以保持连接。

常用命令：`launchctl list | grep com.htma` 查看状态；`launchctl unload ~/Library/LaunchAgents/com.htma.tunnel.plist` 停止隧道。

## 故障排除

### 1. cloudflared tunnel login 提示「Failed to write the certificate」

说明授权在浏览器里成功了，但本机没写入 `~/.cloudflared/cert.pem`（常见于权限或网络）。按提示**浏览器会下载证书**，请：

1. 在**下载目录**里找到刚下载的 `cert.pem`（或类似名称）。
2. 在终端执行（把路径换成你实际下载位置）：
   ```bash
   mkdir -p ~/.cloudflared
   cp ~/Downloads/cert.pem ~/.cloudflared/cert.pem
   ```
3. 再执行：`bash scripts/setup_and_run_cloudflare_tunnel.sh`。

若找不到下载的证书，可改用**不用证书**的方式：用 Cloudflare 控制台给的 **Token** 跑隧道，见下一节。

---

### 2. 不用 cert.pem：用 Token 跑隧道（推荐）

1. 浏览器打开 https://one.dash.cloudflare.com → **网络** → **隧道** → 创建隧道（或使用已有隧道）。
2. 在隧道里添加**公共主机名**：`htma.greatagain.com.cn` → 服务 `http://localhost:5002`。
3. 复制该隧道的**安装命令**里 `--token` 后面的整段 Token。
4. 在本机先启动看板，再跑隧道：
   ```bash
   cd /Volumes/ragflow/hotmaxx/hotmaxxflag
   # 终端1：启动看板
   bash scripts/start_htma.sh
   # 终端2：跑隧道（把 <Token> 换成你复制的）
   bash scripts/run_tunnel_with_token.sh <Token>
   ```
   这样不需要 `cert.pem`，外网照样用 https://htma.greatagain.com.cn 访问。

---

### 3. launchctl 报「Could not find service」或「Input/output error」

- **Could not find service "com.htma.dashboard"**：说明这台 Mac 上**没有安装**看板/隧道的 launchd 服务（没跑过 `scripts/install_launchd_htma.sh`）。  
  - 若只想临时跑：不用 launchctl，在终端里执行 `bash scripts/start_htma.sh` 和隧道脚本即可。  
  - 若要关掉终端也继续跑：先执行一次 `bash scripts/install_launchd_htma.sh`（并按要求创建 `.tunnel-token`），再谈重启服务。
- **Input/output error**：多为权限或路径问题。可用 `launchctl list | grep htma` 看是否已加载；若从未安装过服务，先按上面「没有安装」处理。
