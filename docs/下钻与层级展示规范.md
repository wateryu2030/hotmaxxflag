# 下钻与层级展示规范

本文档约定：各分析模块的**下钻深度**应直至最细数据（如销售日报、毛利明细、单 SKU 销售列表），并在前端用**统一的层级颜色与布局**区分，方便用户识别当前所在层级。

---

## 1. 层级颜色约定

所有带「可展开 / 可下钻」的表格，按层级使用下列样式（对应 CSS class 见 `htma_dashboard/static/index.html`）：

| 层级 | CSS 类 | 含义 | 背景 / 左边框 | 示例 |
|------|--------|------|----------------|------|
| **L0** | `.drill-l0`、`.cat-rank-row`、`.inv-summary`、`.insight-category-row` | 一级汇总 | 蓝 `#38bdf8`，浅蓝底 | 大类、品类汇总、库存大类 |
| **L1** | `.drill-l1`、`.cat-mid-row`、`.insight-brand-row` | 二级 | 青 `#22d3ee`，浅青底 | 中类、品牌 |
| **L2** | `.drill-l2`、`.cat-small-row`、`.insight-subcategory-row` | 三级 | 绿 `#34d399`，浅绿底 | 小类、子品类 |
| **L3** | `.drill-l3`、`.cat-brand-row` | 四级 | 黄 `#fbbf24`，浅黄底 | 品牌（品类排行内） |
| **L4** | `.drill-l4`、`.cat-product-row` | 五级 | 红 `#f87171`，浅红底 | 商品 / SKU |
| **L5** | `.drill-l5`、`.detail-row`、`.category-sale-detail tbody tr` | 最细明细 | 深灰底 `#0f172a`，小字号、灰色字 | 销售日报行、毛利明细行、单 SKU 销售列表 |

- 除 L5 外，每级通过 **左边框 4px 实色** + **浅色半透明背景** 区分。
- L5 为「最细数据」：背景深、字小、无展开图标，表示已到导入的原始数据粒度。

---

## 2. 下钻深度要求

各分析模块应支持**逐级下钻直至最基础信息**：

- **最基础数据**：对应业务上可追溯到「导入的销售日报 / 毛利表 / 库存表」等原始记录，即单日、单 SKU、单品类等明细。
- 若某模块当前只有 1～2 层，后续迭代应补全到可下钻到上述明细（或文档中明确标注「暂不下钻到日报级」及原因）。

| 模块 | 当前下钻路径 | 最细数据 |
|------|--------------|----------|
| **品类排行** | 大类 → 中类 → 小类 → 品牌 → 商品 → 销售明细 | 单日单 SKU 销售（`/api/sale_detail`） |
| **经营分析 - 毛利明细** | 品类汇总 → 展开 | 单日单品类毛利（`/api/profit_detail` 明细行） |
| **经营分析 - 商品明细** | SKU 汇总 → 展开 | 单日单 SKU 销售（`/api/sale_detail`） |
| **库存预警** | 大类 → 展开 | 低库存 SKU 列表（`/api/inv_alert_list`） |
| **消费洞察** | 品类 → 品牌 → 款式(品名) → 货号明细 | 四级看板为单 SKU 销售（品牌、品名、销售额、毛利、销量、毛利率、折扣%） |

---

## 3. 前端实现要点

- **复用 class**：新做或调整带层级的表格时，优先使用 `.drill-l0`～`.drill-l5`，或已有语义 class（如 `cat-rank-row`、`insight-brand-row`），以保持颜色一致。
- **展开图标**：非 L5 的可展开行使用 `.expand-icon`，展开后旋转 90°。
- **悬停**：各层级行 `:hover` 时可用略深背景（如 `#334155`）提示可点击，但不改变层级色。
- **嵌套容器**：下钻插入的「整块表格」容器（如 `.cat-rank-mid-wrap`、`.cat-brands-wrap`）使用略深背景（如 `rgba(15,23,42,0.5)`）与主表区分。
