# 看板「数据缺失」提示说明与处理

## 一、提示来源与含义

| 提示 | 含义 | 常见原因 |
|-----|------|----------|
| **库存总额: 0** | 当前没有库存数据或库存表为空 | 本次导入**未包含库存 Excel**（只导了销售日报+销售汇总） |
| **共 76 天数据** | 所选 KPI 周期内，有销售记录的**自然日**只有 76 天 | 库里销售日期范围为 2025-12-21 ~ 2026-03-06，约 76 天；若 KPI 选了一年（如 2025/03/06–2026/03/06），就会显示「共76天数据」 |
| **上期 销售额: 0.00** | 对比的「上期」区间内没有销售数据 | 上期一般为去年同期（如 2024-03-05~2025-03-05），库里没有该时间段数据 |
| **需至少366天日线数据方可计算同比** | 按日粒度做同比需要至少 366 个有数据的自然日 | 目前只有约 76 天数据，无法算同比，属正常提示 |
| **周几对比缺某些天** | 周几对比图里部分星期没有柱子 | 在 76 天内，若某些星期几没有销售记录，就不会显示该天 |

---

## 二、怎么解决

### 1. 库存总额为 0

- **原因**：导入时下载目录里**没有「库存」或「库存查询」类 Excel**，或文件名/格式未被识别。
- **处理**：
  - 把**库存/库存查询** Excel 放到 `~/Downloads`（或你用的下载目录），再执行：
    ```bash
    bash scripts/run_auto_import.sh ~/Downloads --multi
    ```
  - 或在看板「数据导入」页**单独上传库存文件**（先传销售、再传库存，避免超时）。

### 2. 只有 76 天数据 / 上期为 0 / 无法算同比

- **原因**：库里目前只有约 2025-12-21 ~ 2026-03-06 的销售数据，没有更早历史。
- **处理（二选一或都做）**：
  - **改 KPI 周期**：在看板把「起止日期」改成**实际有数据的区间**（如 2025-12-21 至 2026-03-06），则「共 X 天数据」会与真实天数一致，环比也会基于该区间计算。
  - **补历史数据**：若有更早的销售日报/销售汇总，放入下载目录后再次执行 `run_auto_import.sh ... --multi`，数据会增量写入；数据覆盖到一年后，同比即可计算。

### 3. 周几对比缺周一/周三/周五等

- **原因**：在现有 76 天里，这些星期几可能没有销售记录，或该周期内本身就没有那几天。
- **处理**：属正常现象。补更多日期的销售数据后，周几分布会更完整。

---

## 三、导入结果如何自检

导入完成后，终端会输出类似：

- `销售表: XXXXX 条，销售金额合计: ...`
- `库存表: XXXXX 条，库存金额合计: ...`
- `日期范围: YYYY-MM-DD ~ YYYY-MM-DD`

若**库存表: 0 条**，说明本次没有导入任何库存文件，看板「库存总额」会为 0，按上面「库存总额为 0」处理即可。
