# 数据导入逻辑说明

## 1. 销售数据：日报 vs 汇总

- **销售日报**：按日、按 SKU 的明细，导入到 `t_htma_sale`。同一 (日期, 货号) 在**本批**内先按行聚合再写入，同键**覆盖**不累加。
- **销售汇总**：同样是按日、按 SKU 的汇总表，也写入 `t_htma_sale`。  
  - **重要**：销售汇总导入时，对同一 (日期, 货号) **始终覆盖、不累加**（`overwrite_on_duplicate=True`）。  
  - 这样无论是「日报 + 汇总一起传」还是「只传汇总」或「先导日报再导汇总」，都不会在已有数据上再累加，避免某日数据翻倍（如 3 月 7 日重复）。

## 2. 3 月 7 日重复的原因（已修复）

- **原因一**：之前销售汇总在「本批没有销售日报」时使用**累加**模式。若先导过销售日报（含 3 月 7 日），再单独导销售汇总，同 (日期, 货号) 会再加一遍 → 销售额翻倍。  
- **修复**：销售汇总统一改为**覆盖**模式（`import_sale_summary(..., overwrite_on_duplicate=True)`），在网页上传 `api_import` 和脚本 `auto_import_from_downloads` 中均已改为始终传 `True`。
- **原因二**：Excel 中带有「汇总数据」「合计」「小计」等汇总行，若未被识别会当明细导入，造成重复或虚高。  
- **修复**：在 `import_logic.py` 中加强汇总行识别：  
  - `SALE_SUMMARY_ROW_KEYWORDS` / `SUMMARY_SUBSTRINGS` 增加「汇总数据」；  
  - 货号为纯 0（如 `0`、`000000`）且品类或品名含合计/小计/汇总时，视为汇总行跳过。

## 3. 汇总行过滤规则

- 以下行**不会**写入销售/毛利/库存表，仅作跳过：  
  - 货号/品类/品名在关键词集合中：`总计、合计、小计、求和项、汇总、合计行、总计行、小计行、货号、汇总数据`；  
  - 货号/品类/品名**包含**：合计、总计、小计、汇总、求和项、合计行、总计行、小计行、汇总数据；  
  - 货号为纯数字 0（如 `0`、`000000`）且品类或品名包含上述汇总类字样；  
  - 货号列出现「求和项:…」「总计」「合计」等导出统计字样。  
- 实现位置：`import_logic.py` 中 `SALE_SUMMARY_ROW_KEYWORDS`、`SUMMARY_SUBSTRINGS`、`_is_sale_summary_row()`。

## 4. 各块数据与表

| 上传块 / 脚本识别 | 写入表 | 说明 |
|------------------|--------|------|
| 销售日报         | `t_htma_sale` | 按 (日期, 货号) 聚合后覆盖 |
| 销售汇总         | `t_htma_sale` | 按 (日期, 货号) **覆盖**，不累加 |
| 实时库存         | `t_htma_inv`  | 按 (日期, 货号) 覆盖 |
| 品类附表         | `t_htma_category` | 清空后全量写入（清空前有确认） |
| 人力 / 商品档案  | 不在网页上传白名单中，仅各自导入入口处理 |

## 5. 入口

- **网页**：导入页上传 Excel，仅处理 `IMPORT_ALLOWED_KEYS`（销售日报、销售汇总、实时库存、品类附表），不触碰人力、商品档案。  
- **脚本**：`scripts/run_auto_import.sh [下载目录] [--multi]`，从目录识别销售日报/销售汇总/实时库存文件并调用 `import_logic` 对应函数；销售汇总同样使用覆盖模式。

## 6. 修正 3 月 7 日已重复数据（本地执行）

若库中 2026-03-07 已翻倍，在本地执行以下任一方式即可按一半还原：

**方式一：MySQL 客户端**

```bash
mysql -h 主机 -u 用户 -p 数据库名 < scripts/fix_sale_20260307_double.sql
```

**方式二：Python（使用项目 .env 配置）**

```bash
cd 项目根目录
python3 scripts/run_fix_sale_20260307.py
```

（需已安装 pymysql，与看板使用同一 DB 配置。）

## 7. 数据入口与出口一致（统计口径）

- **单一数据源**：所有看板展示（总销售额、总毛利、环比、销售与毛利趋势、周几对比）均**只从 `t_htma_sale` 聚合**，与手工统计、验证脚本口径一致。
- **导入后同步**：只要有销售日报或销售汇总导入（网页上传或脚本导入），都会自动执行 `refresh_profit(conn)`，将 `t_htma_sale` 按日+品类汇总写入 `t_htma_profit`，供导出/分账等仍读毛利表的逻辑使用。
- **校验**：`scripts/verify_sale_consistency.py [start_date] [end_date]` 校验同一周期下「总汇总 = 按日相加 = 按周几相加」；`scripts/openclaw_verify_sale_consistency.sh` 可再校验 KPI/趋势/周几对比 三个接口与库一致（需服务已启动）。
