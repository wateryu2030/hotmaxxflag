# 飞书自建应用机器人（群内对话）配置说明

使用与**飞书扫码登录**同一套自建应用（`FEISHU_APP_ID` / `FEISHU_APP_SECRET`），在飞书开放平台开启**机器人**与**事件订阅**后，群成员 **@ 机器人** 或 **与机器人私聊** 时，看板服务会收到事件并自动回复。

**能力说明（已实现）**：机器人会读取本地 **`config/feishu_bot/soul.md`**（话术）、**`command_map.json`**（关键词）、**`config.yaml`**（功能开关与日志路径），支持 **项目介绍**、**核心表数据概览（只读 COUNT）**、**高库存示例查询（参数绑定，非拼接 SQL）**。详细设计见同目录 **`sql_templates.sql`**（说明用）。**不会**执行用户输入的任意 SQL。

回调路径（需公网 HTTPS，与 `HTMA_PUBLIC_URL` 一致；**开放平台里填写的 URL 须与下列之一完全相同**）：

```text
https://你的域名/api/feishu/bot/event
```

或与常见教程一致的别名（同一进程、同一端口 **5002**，非独立 5000 服务）：

```text
https://你的域名/feishu/callback
```

示例：`https://htma.greatagain.com.cn/api/feishu/bot/event` 或 `https://htma.greatagain.com.cn/feishu/callback`。

**反代 / 穿透**：将 HTTPS 转发到本机 **`http://127.0.0.1:5002`**（看板 `app.py` 监听端口），路径原样转发，勿映射到不存在的 5000 端口。

**事件 JSON 格式**：实现已同时兼容 **2.0 订阅**（`schema: "2.0"` + `header.event_type`）与 **旧版**（`type: "url_verification"` / `type: "event_callback"`），地址校验均返回合法 JSON（如 `{"challenge":"..."}`）。发消息使用 **`tenant_access_token`**（`/auth/v3/tenant_access_token/internal`），与飞书自建应用常规做法一致。

## 一、开放平台操作（hotmaxxblog / 你的自建应用）

1. 打开 [飞书开放平台](https://open.feishu.cn/app) → 进入对应应用。
2. **添加应用能力** → 开启 **机器人**。
3. **权限管理** → 开通并发布版本（按需搜索并勾选），常见包括：
   - 以应用身份读取通讯录（若需）
   - **接收群聊中 @ 机器人** 的消息（`im:message.group_at_msg` 或控制台展示的同义能力）
   - **读取用户发给机器人的单聊消息**（私聊）
   - **发送消息**（`im:message` / `im:message:send_as_bot` 等，以控制台为准）
4. **事件订阅**：
   - **请求地址 URL**：填上文的 `https://.../api/feishu/bot/event`。
   - 复制控制台的 **Verification Token** → 写入项目根 `.env` 的 `FEISHU_VERIFICATION_TOKEN`（**强烈建议**，用于校验 `header.token`，防止伪造请求）。
   - 若开启 **Encrypt Key（加密）**：将密钥写入 `.env` 的 `FEISHU_ENCRYPT_KEY`，并确保服务器已安装 `pycryptodome`（已在 `htma_dashboard/requirements.txt` 中）。
   - 订阅事件：**消息与群组** → **`im.message.receive_v1`**（接收消息 v2.0）。
5. 保存后点击 **验证**，飞书会向上述 URL 发起 **URL 验证**；看板需已部署且路由可访问，验证才会通过。
6. **创建版本** → **申请发布**（企业内可用即可），否则机器人无法在群里正常使用。
7. 在目标 **群聊** → 设置 → 群机器人 / 添加应用 → 添加本应用机器人。

## 二、项目环境变量（`.env`）

与登录共用：

- `FEISHU_APP_ID` / `FEISHU_APP_SECRET`：应用凭证（已配置则可不改动）。

机器人专用（可选）：

| 变量 | 说明 |
|------|------|
| `FEISHU_VERIFICATION_TOKEN` | 与事件订阅里的 Verification Token **完全一致**；配置后校验每条事件的 `header.token`。 |
| `FEISHU_ENCRYPT_KEY` | 与事件订阅加密密钥一致；仅当控制台开启加密时必填。 |
| `FEISHU_BOT_REPLY_MODE` | `mention`（默认）：仅 **群聊中 @ 机器人** 时回复；私聊始终会回复。`all`：群聊里任意文本都回复（易刷屏）。`p2p_only`：仅私聊回复，群聊不回复。 |
| `FEISHU_BOT_REPLY_PREFIX` | 回复前缀，默认 `【好特卖】`。 |
| `FEISHU_BOT_OPEN_ID` | 一般留空；自动调用 `bot/v3/info` 获取机器人 `open_id`。若识别 @ 异常可手动填写。 |
| `FEISHU_BOT_DB_ALLOWED_OPEN_IDS` | **数据库类指令白名单**：飞书用户 `open_id` 逗号分隔；未配置则拒绝「数据概览」「库存大于N」等。填 `*` 表示不校验（**仅内网测试**，生产禁用）。 |
| `FEISHU_BOT_MYSQL_USER` / `FEISHU_BOT_MYSQL_PASSWORD` | 可选：机器人专用 **MySQL 只读账号**；未设置则沿用 `MYSQL_USER` / `MYSQL_PASSWORD`。 |
| `FEISHU_BOT_DB_REQUIRE_ALLOWLIST` | 默认 `1`：必须配置白名单才允许库查询；设为 `0` 且无白名单时放开（不推荐）。 |

**本地配置文件（可改文案与关键词）**：`config/feishu_bot/soul.md`、`config/feishu_bot/command_map.json`、`config/feishu_bot/config.yaml`。日志默认：`logs/feishu_bot.log`（按配置轮转）。

**指令示例**：`项目介绍`、`数据概览`、`库存大于100`、`高库存`（默认按阈值 100）。

修改 `.env` 后需 **重启看板进程**。

安装依赖（若尚未安装加密库）：

```bash
cd /Volumes/ragflow/hotmaxx/hotmaxxflag
.venv/bin/pip install pycryptodome
```

## 三、行为说明

- 默认 **`FEISHU_BOT_REPLY_MODE=mention`**：群里需 **@ 机器人** 才会处理消息；**私聊**始终处理。
- 回复内容由 **`feishu_bot_brain.build_reply`** 生成：匹配关键词后返回话术或只读库结果；未匹配则返回 `soul.md` 中的兜底说明。

## 四、故障排查

| 现象 | 处理 |
|------|------|
| URL 验证失败 | 确认公网能 POST 到该 URL；无反代路径改写；HTTPS 证书有效；看板已重启。 |
| 验证通过但发消息无反应 | 是否已 **发布应用版本**；群是否 **添加该机器人**；群里是否 **@ 机器人**（`mention` 模式）。 |
| 403 invalid signature / forbidden | 核对 `FEISHU_ENCRYPT_KEY`、`FEISHU_VERIFICATION_TOKEN` 与控制台是否一致；改后重启看板。 |
| 解密失败 | 安装 `pycryptodome`；加密密钥勿含多余空格。 |

## 五、与「自定义机器人 Webhook」的区别

- **`FEISHU_WEBHOOK_URL`**：仅 **向群推送** 文本，**不能**在群内接收用户消息并对话。
- **本方案（事件订阅 + 自建应用机器人）**：可 **收消息并回复**，适合群内问答场景。
